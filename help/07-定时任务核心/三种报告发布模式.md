# 三种报告发布模式 / Report Publish Modes

### 中文

定时任务生成深度简报后，可将报告以网页形式发布。在任务列表上方的 **发布设置** 中配置全局通道。

## 1. 默认通道（官方托管）

- **说明**：使用官方安全发布通道  
- **优点**：零配置；页面排版适合手机阅读与转发  

## 2. 自定义通道（自建托管）

- **说明**：简报数据托管在你自己的服务上  
- **配置**：自建地址 + 安全密钥  
- **优点**：数据自有；可设保留天数（如 180 天自动清理）  

## 3. SEO 博客门户

- **说明**：把精选报告推送到独立站点 / 博客  
- **配置**：站点 URL + 安全密钥；可映射分类、专栏、作者  
- **示例**：[blog.oinchain.com](https://blog.oinchain.com)（由 RSSFlow 全自动整理更新）  

## 标签合并

发布时自动合并三类标签，便于检索：

1. AI 设置中的全局分类标签  
2. 该定时任务的筛选标签  
3. 发布配置中的默认附加标签  

### English

After a workflow compiles a briefing, publish it as a web page via **Publish Settings**.

## 1. Default (Official Hosting)

Zero-config, mobile-friendly pages for reading and sharing.

## 2. Custom (Self-Hosted)

Host on your own infrastructure with URL + security key. Control retention (e.g. purge after 180 days).

## 3. Blog Portal (SEO Site)

Push reports to your site with categories, series, and author mapping.  
Demo: [blog.oinchain.com](https://blog.oinchain.com).

## Tag Merging

On publish, tags merge from:

1. Global AI classification tags  
2. Workflow filter tags  
3. Default publication tags
