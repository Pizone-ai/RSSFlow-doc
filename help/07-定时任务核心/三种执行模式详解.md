## 三种执行模式详解 / Execution Modes

### 中文

RSSFlow Pro 提供了三种灵活的自动化执行工作流模式，帮助您按需处理和聚合文章。您可在工作流编辑器中进行切换。

**1. 单指令分析 (Single)**
- **怎么用**：选择一个 AI 指令（例如“每日简报”），系统会自动把抓取到的多篇文章整理好，调用该指令一次性生成一份合并的总结报告。
- **适用场景**：简单的资讯汇总、今日财经要闻简报。

**2. 串行链式流 (Sequential Chain)**
- **怎么用**：像流水线一样设置多个步骤（步骤一 → 步骤二）。第一步的 AI 分析结果会自动作为下一步的参考输入。
- **适用场景**：需要多步骤深加工的情报。例如：步骤一“提取文章要点”，步骤二“基于提取的要点撰写一篇中文行业综述”。

**3. 并行汇总流 (Split-Merge)**
- **怎么用**：采用分头分析再合并的模式。
  - **分析阶段**：AI 会从不同维度（例如同时配置“技术影响分析”和“市场商机评估”两个维度）并行独立分析所有的文章。
  - **汇总阶段**：最后由汇总模型将各个维度的独立分析结果合并，生成一份视角的决策报告。
- **适用场景**：多角度的深度交叉分析与研报撰写。

---

### English

RSSFlow Pro provides three flexible workflow execution modes for processing and compiling article insights. You can configure these modes directly within the workflow editor.

**1. Single Mode**
- **How it works**: Select a single AI command (e.g., "Daily Briefing"). The system aggregates all fetched articles and generates a single compiled summary report.
- **Best for**: Simple daily roundups and quick newsletters.

**2. Sequential Chain Mode**
- **How it works**: Set up sequential pipeline steps (Step 1 → Step 2). The output of the previous AI step is automatically fed into the next step as variables for further polishing, translating, or expanding.
- **Best for**: Multi-stage information processing, e.g., Step 1: "Extract raw facts" → Step 2: "Write an opinion post based on these facts."

**3. Split-Merge Mode**
- **How it works**:
  - **Split Phase (Map)**: Analyzes the aggregated articles from multiple dimensions in parallel (e.g., analyze "Tech Impact" and "Market Value" concurrently).
  - **Merge Phase (Reduce)**: Combines all parallel dimensional outputs into a single, comprehensive final synthesis report.
- **Best for**: Complex, multi-perspective investigations and professional research reports.
