2:Tb6b,<h2>AI 智能</h2>
<p><strong>Q：新文章为什么没有自动摘要？</strong></p>
<ol>
<li><strong>总开关</strong>：AI 设置页顶部 <strong>启用 AI 功能</strong> 已开启</li>
<li><strong>默认模型</strong>：Basic 已验证，并已 <strong>设为当前启用</strong></li>
<li><strong>订阅源</strong>：该源的 <strong>Auto AI</strong> 已打开；未开则不会自动摘要</li>
</ol>
<p><strong>Q：验证模型一直超时？</strong></p>
<p>测试超时约 20 秒。常见原因：</p>
<ul>
<li>API Key 拼写错误</li>
<li>API Host 不可达（部分境外服务需网络或兼容中转，如 SiliconFlow）</li>
<li>模型名称不正确；可点刷新拉取模型列表</li>
</ul>
<p><strong>Q：获取模型列表失败？</strong></p>
<p>先确认 Key 与 Host 正确；部分供应商需先能连通，才会返回模型目录。</p>
<hr />
<h2>订阅源</h2>
<p><strong>Q：更新经常失败？</strong></p>
<ul>
<li>在浏览器直接打开 RSS URL，确认仍是有效 XML</li>
<li>部分外网源可能超时</li>
<li>使用列表中的 <strong>手动同步</strong> 重试</li>
</ul>
<p><strong>Q：OPML 导入失败？</strong></p>
<p>确认文件为标准 OPML / XML，编码建议 UTF-8。</p>
<hr />
<h2>消息推送</h2>
<p><strong>Q：Telegram 收不到？</strong></p>
<ul>
<li>Bot Token 与 Chat ID 是否正确（群组 ID 常带负号 <code>-</code>）</li>
<li><strong>必须先</strong>在机器人对话中点 <strong>Start</strong> 或发送 <code>/start</code></li>
<li>推送设置中对应类别是否勾选</li>
</ul>
<p><strong>Q：飞书机器人收不到？</strong></p>
<ul>
<li>Webhook 是否完整复制</li>
<li>若飞书开启签名校验，Secret 必须一致；未开启则 Secret 留空</li>
</ul>
<hr />
<h2>定时任务</h2>
<p><strong>Q：到点不执行？</strong></p>
<ul>
<li>任务开关是否开启</li>
<li>深度报告默认用 <strong>复杂模型 (Advanced)</strong>，请确认已配置</li>
<li>过滤条件是否过严，导致窗口内无文章</li>
<li>查看执行历史中的错误</li>
<li>未激活版本最多约 2 个活跃任务</li>
</ul>
<p><strong>Q：执行成功但推送内容为空？</strong></p>
<ul>
<li>任务内是否关联并启用了推送通道</li>
<li>在执行历史中预览报告，确认模型是否生成了正文</li>
</ul>
<hr />
<h2>数据与恢复</h2>
<p><strong>Q：如何恢复默认？</strong></p>
<p>各面板（如标签范围）有 <strong>恢复默认值</strong>。若要清空全部数据，可卸载扩展后重装。</p>
<p><strong>Q：数据存在哪里？会上传吗？</strong></p>
<p>采用 <strong>本地优先</strong>：文章、摘要、图谱等多在浏览器本地（IndexedDB / 本地存储）。 只有在你主动配置的 AI / 推送 / 云门户 / 激活等场景才会发出必要数据。</p>
