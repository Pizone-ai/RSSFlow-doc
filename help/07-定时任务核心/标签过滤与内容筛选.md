# 标签过滤与内容筛选 / Filters & Content Selection

### 中文

当您创建自动化定时任务时，RSSFlow Pro 允许您通过精细的内容过滤器筛选出高价值资讯作为任务输入，防止无关信息噪音干扰您的简报。

**过滤器可配置维度：**
您可以在工作流编辑器的“内容筛选”面板中调整以下参数：
- **时间范围**：仅处理最近特定时间内的文章（如最近 24 小时、48 小时或 7 每天）。
- **指定订阅源**：您可以将任务绑定到特定的 RSS 订阅源，或选择处理所有源。
- **标签过滤**：只处理带有特定 AI 自动分类标签的文章（支持输入多个标签）。
- **关键词过滤**：只包含包含特定关键词的文章（支持设置多个词）。
- **最低 AI 评分**：限定文章的 AI 评分阈值（例如只汇总评分 ≥ 80 分的优质深度文章）。
- **文章数量上限**：限制单次任务处理的最大文章总数（默认 100 篇），防止消耗过量 Token。

**筛选过滤的工作逻辑：**
系统获取并筛选文章时，会自动经过以下多层过滤漏斗：
1. **基础筛选**：
   - 提取您指定时间范围内的全部文章。
   - 检查文章是否已经生成了 AI 摘要（为了保证简报质量，系统默认只汇总已由 AI 摘要分析过的文章）。
   - 匹配订阅源与最低 AI 评分要求。
2. **标签匹配规则**：
   - 系统将文章的 AI 分类标签与您在任务中配置的过滤标签进行不区分大小写的比对。
   - 支持多标签匹配（只要满足您设定的任意一个标签，即可通过筛选）。
3. **关键词匹配规则**：
   - 系统会在文章的摘要、要点和关键词中搜索您设定的过滤词。
   - 同样支持多关键词匹配（满足其中任意一个词即可）。

**高级搜索（SQLite 语法）：**
如果您有更为复杂的筛选需求，可以使用“高级 SQLite 查询”模式。在该模式下，您可以利用逻辑运算符组合出强大的检索条件：
- 用**空格**或 **`AND`** 表示同时满足两个词（例如：`AI 芯片` 表示必须同时包含这两个词）。
- 用 **`OR`** 表示满足其中之一。
- 用 **`NOT`** 排除包含特定词的文章（例如：`智能手机 NOT 苹果`）。
- **智能降级保护**：当高级检索发生异常或处理时间过长时，系统会自动安全降级为标准模式进行常规检索，确保您的定时任务永远不会中断。

### English

When creating automation workflows, RSSFlow Pro offers powerful content filters so you can target high-value articles and block irrelevant noise.

**Available Filter Dimensions:**
You can configure the following filters in the "Content Filter" panel of the workflow editor:
- **Time Range**: Process articles received within a specific timeframe (e.g., last 24 hours, 48 hours, or 7 days).
- **Feeds Filter**: Bind the workflow to specific RSS subscriptions or process all articles.
- **Tags Filter**: Target articles with specific AI-generated category tags (supports multiple tags).
- **Keywords Filter**: Process only articles containing specified keywords.
- **Minimum AI Score**: Select only high-value deep-read articles (e.g., only summarize articles rated 80/100 or higher).
- **Max Article Limit**: Limits the maximum number of articles processed in a single run (default is 100) to control API token usage.

**Filter Execution Flow:**
The system filters information step-by-step:
1. **Base Matching**:
   - Collects articles within the specified time range.
   - Ensures the article already has an AI summary (by default, workflows process only AI-summarized articles to maintain briefing quality).
   - Verifies feed source matches and filters out articles below the minimum AI score.
2. **Tag Matching Logic**:
   - Automatically compares the article's AI tags with your task filter tags (ignoring `#` prefixes and capitalization).
   - If multiple tags are configured, matching any one of them (OR logic) qualifies the article.
3. **Keyword Matching Logic**:
   - Scans the AI summary, key points, and keyword lists for matches.
   - If multiple keywords are specified, matching any one of them qualifies the article.

**Advanced SQLite Querying:**
For power users who need advanced logic, the filter supports an Advanced Search mode using classic search operators:
- Use **Spaces** or **`AND`** to match both words (e.g., `AI Chips`).
- Use **`OR`** to match either word.
- Use **`NOT`** to exclude specific keywords (e.g., `Smartphone NOT Apple`).
- **Safety Fallback**: If an advanced search takes too long or fails, the system automatically degrades to standard indexed search, ensuring your scheduled workflows run without interruption.
