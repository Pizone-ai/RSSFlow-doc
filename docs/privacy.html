隐私政策

最后更新：2026-07-14

“RSSFlow Reader（以下简称“RSSFlow”或“本扩展”）重视您的隐私与数据安全。本隐私政策用于说明：当您安装、使用本扩展时，我们会处理哪些数据、这些数据如何被存储、在什么情况下会被传输，以及您可如何管理自己的数据。”

1. 简介

RSSFlow 的核心设计原则是：尽可能将数据保存在用户本地设备中。但当您主动启用某些联网功能时，相关数据可能会被发送至您选择的第三方服务，或发送至我们提供的功能服务基础设施，以完成对应功能。

2. 我们处理哪些数据

2.1 本地阅读与订阅数据

这些数据通常保存在您的浏览器本地，用于实现 RSS 阅读和管理功能：

  • 您添加的 RSS 订阅源地址（RSS Feed URL）
  • 文章标题、链接、摘要、正文缓存或解析后的文本内容
  • 已读/未读状态、收藏状态、标记状态、标签、筛选偏好
  • 发现页、摘要页、Flow 视图等功能产生的本地缓存数据

2.2 设置与个性化配置

为了提供个性化体验，本扩展可能会在本地保存：

  • 主题、语言、显示方式、阅读偏好
  • 快捷指令、自定义命令、自定义提示词（Prompt）
  • AI 功能开关、模型选择、参数配置
  • TTS 语音播放设置、自动化任务配置、通知配置
  • MCP Bridge 配置等扩展功能设置

2.3 AI 与聊天相关数据

如果您启用了 AI 摘要、AI 聊天、HTML 预览生成或其他 AI 相关能力：

  • 文章正文、摘要上下文、提问内容
  • 您输入的 Prompt、指令、聊天消息及其生成结果
  • 您配置的 API Host、模型名称、参数及 API Key 等访问凭据

2.4 激活、试用与权限校验

  • 本地生成的设备标识符或用户标识符
  • 激活状态、试用状态、权限及到期时间信息
  • 激活码及签名校验信息

2.5 通知与推送相关数据

如果您启用了 Telegram、飞书或其他通知/自动化推送能力：

  • Telegram Bot Token, Chat ID
  • 飞书 Webhook 地址及相关配置
  • 被推送的文章标题、链接、摘要、自动化分析结果
  • 自动化任务名称、报告链接、执行结果

2.6 语音播报与 TTS 数据

如果您启用了语音播报或云端 TTS 功能：

  • 待朗读的文本内容
  • TTS 语音配置
  • 生成后的音频缓存数据

3. 浏览器权限使用说明

为了提供离线阅读、AI 智能分析以及 HTML 解析等核心功能，RSSFlow 会申请并使用以下浏览器权限：

  • offscreen (后台离线文档)：在 Manifest V3 规范下，后台 Service Worker 缺少 DOM 环境。我们通过创建离线文档来安全地解析 HTML/XML 格式的订阅源，并运行本地 SQLite WebAssembly 数据库引擎。
  • unlimitedStorage (无限制本地存储)：用于在本地安全地存储离线文章、AI 对话历史、向量索引等数据。该权限可防止数据量超出浏览器默认的 5MB 配额时出现数据丢失或本地数据库损坏。

4. 数据如何被存储

4.1 本地存储

RSSFlow 的大多数数据默认保存在您的设备本地（如浏览器本地存储或本地数据库），包括订阅记录、设置、聊天会话、自动化配置等。

4.2 不会默认同步至服务器

RSSFlow 不会默认把您本地保存的全部阅读数据或设置数据上传到我们的服务器。仅在您主动启用依赖联网的功能时才会发送必要数据。

5. 数据在什么情况下会被传输

5.1 获取 RSS 内容更新

当您添加 RSS 源后，本扩展会向对应的 RSS 源站点发起请求，以拉取订阅内容。

5.2 使用 AI 摘要、AI 聊天或 AI 生成能力时

当您启用 AI 功能时，文章内容、提问及其参数可能会发送至您选择的 AI 服务提供商（OpenAI, Gemini, SiliconFlow, DeepSeek 等）。

5.3 使用激活、试用或权限验证功能时

设备标识符、激活码等信息可能会发送至我们的激活服务以完成授权。

5.4 使用 Telegram、飞书等消息推送功能时

文章摘要、链接等内容可能会发送至您配置的消息接收端。

5.5 使用 MCP Bridge、远程桥接或自动化能力时

输入指令、文章上下文及任务结果可能会传输以完成工作流执行。

5.6 使用云端 TTS 能力时

待朗读文本可能会被发送到相应语音服务用于生成音频。

6. 我们如何共享数据

我们不会出售您的个人数据。数据仅在您启用特定功能时共享给以下方：

  • RSS 内容来源站点
  • 您主动选择或配置的 AI 服务提供商
  • 您配置的消息接收平台（Telegram, 飞书等）
  • 我们提供的功能基础设施（如权限校验、MCP Bridge、报告上传、TTS 中转）

6.5 第三方服务的独立规则

这些第三方将依据其各自的隐私政策处理数据。我们建议您在启用相关功能前先查阅对应政策。

7. 数据保留期限

7.1 本地数据

保存在本地的数据会一直保留直到您主动删除、重置扩展、卸载扩展或浏览器清理存储。

7.2 发送到第三方或远程服务的数据

保留时间取决于对应服务完成处理所需时间及第三方服务自己的存储策略。

8. 数据安全

我们通过本地存储、HTTPS 加密、签名校验等措施保护数据。但请妥善保管您的 API Key、激活码、Token 等敏感凭据。

9. 您的权利与控制方式

您保留对数据的完全控制权：

  • 查看、修改或通过重置功能清除本地数据
  • 关闭 AI、推送、桥接、TTS、自动化等联网功能
  • 通过卸载扩展删除大部分本地存储内容

10. 未成年人隐私

本扩展并非专门面向未成年人设计，请在监护人指导下使用。

11. 隐私政策的更新

随产品迭代或法规变化，我们可能会更新本政策。更新版本将在本页面发布并标注日期。
