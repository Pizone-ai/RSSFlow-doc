# 模型提供商配置 / Provider Configuration

### 中文

配置大模型供应商后，才能使用摘要、热点、对话等 AI 能力。

## 入口

**设置 → AI 设置**

## 总开关

页面顶部 **启用 AI 功能** 关闭时，AI 配置与相关功能均不可用。

## 两套配置

| 配置 | 适用场景 | 建议模型类型 |
|------|----------|--------------|
| 默认模型 (Basic) | 自动摘要、打标、评分等高频轻量任务 | 快、便宜、稳定 |
| 复杂模型 (Advanced) | 深度对话、研报、热点聚合等 | 强推理、大上下文 |

## 支持的供应商

- **DeepSeek** — 高性价比推理与聊天  
- **OpenAI** — GPT 系列  
- **Anthropic Claude** — 长文逻辑与研报风格  
- **Google Gemini** — 大上下文  
- **OpenAI Compatible** — 兼容中转 / 聚合（如 SiliconFlow、火山引擎）  
- **Local LLM** — Ollama / LM Studio 等本机模型  

## 参数说明

| 字段 | 说明 |
|------|------|
| API Key | 供应商密钥（密码框隐藏） |
| API Host | 默认官方地址；中转可改写 |
| Model Name | 模型代号；可手动输入或刷新列表选择 |

## 验证与生效

1. 填写参数后点 **验证并暂存配置**（后台连通性测试）。  
2. 通过后点 **设为当前启用供应商** — **只有这一步后配置才真正生效**。

## 本地模型

选择 Local LLM 时，按页面内跨域指引操作：

- Ollama：允许 `chrome-extension://*` 来源  
- LM Studio：开启 CORS  

更完整的步骤见：**快速开始 → AI 密钥配置**。

### English

Configure a provider before using summaries, discovery, or chat.

## Access

**Settings → AI Settings**

## Master Switch

**Enable AI Features** must be on.

## Two Profiles

| Profile | Use for | Prefer |
|---------|---------|--------|
| Basic | Auto summary, tags, scores | Fast / cost-efficient |
| Advanced | Deep chat, reports, clustering | Strong reasoning / large context |

## Providers

DeepSeek, OpenAI, Anthropic Claude, Google Gemini, OpenAI-compatible aggregates, and Local LLM (Ollama / LM Studio).

## Fields

| Field | Notes |
|-------|-------|
| API Key | Masked token from your provider |
| API Host | Official default; override for proxies |
| Model Name | Type or refresh the list |

## Validate & Activate

1. **Validate & Stash** — connectivity test  
2. **Set as Active Provider** — required for the profile to take effect  

## Local Models

Follow the in-app CORS guide for Ollama / LM Studio.  
See **Quick Start → AI Key Configuration** for full steps.
