<h2>怎么用（基础）</h2>
<ol>
<li>先保证相关文章已经进了当前对话（例如：信息流里某分组的「AI 聊天」，或侧边栏打开 AI 聊天后再带入上下文）。</li>
<li>在聊天输入区打开 <strong>指令菜单 / 指令中心</strong>。</li>
<li><strong>点一条指令</strong> → AI 立刻用该角色处理当前文章，你一般<strong>不用再手打长提示词</strong>。</li>
<li>等输出结束后，可以继续追问，例如：「把第三条展开」「用表格对比」。</li>
</ol>
<p><strong>小建议：</strong> 相关订阅源先打开 <strong>自动 AI 摘要</strong>，再跑专家指令，通常更稳、也更省费用。</p>

<h2>一、全部 22 条指令介绍（请先读完这一节）</h2>
<p>下面按编号列出全部内置指令。每条都写清：<strong>它是干什么的、什么时候该点它、大概会得到什么</strong>。</p>
<h3>1. 创意构思者（Ideator）</h3>
<p>从你当前加载的多篇文章里，找出「可以继续做内容」的话题方向，把零散新闻收成几个<strong>话题簇</strong>，并给出可写的角度。</p>
<ul>
<li><strong>什么时候用：</strong> 不知道写什么、要做周选题、要给频道定主题。</li>
<li><strong>你会得到：</strong> 选题清单、话题分组、每个方向可以怎么切入的建议。</li>
<li><strong>注意：</strong> 它偏「找题」，不是直接写成终稿；成稿请再配草稿构建器。</li>
</ul>
<h3>2. 趋势筛选器（Trend Selector）</h3>
<p>在已有资讯里判断哪些话题<strong>更热、更值得优先跟</strong>，按传播潜力或时效性做排序，并简单说明理由。</p>
<ul>
<li><strong>什么时候用：</strong> 信息很多、只想抓今天该盯的 3～5 个热点。</li>
<li><strong>你会得到：</strong> 热点排序、为何重要、哪些可以先放一放。</li>
<li><strong>注意：</strong> 依据的是你订阅流里的内容，不是全网热搜榜。</li>
</ul>
<h3>3. 草稿构建器（Draft Builder）</h3>
<p>把多篇素材整理成<strong>一篇结构清楚的文章骨架或初稿</strong>（标题、小标题、段落逻辑）。</p>
<ul>
<li><strong>什么时候用：</strong> 素材已经够了，要开始写深度文、周报、综述。</li>
<li><strong>你会得到：</strong> 大纲，以及可继续改的正文草稿。</li>
<li><strong>注意：</strong> 初稿可能偏「正确但平」；对外发布前常用「人性化改写」润一遍。</li>
</ul>
<h3>4. 人性化改写（Humanizer）</h3>
<p>专门改语气：减少「作为 AI」「首先其次最后」等机械感，让文字读起来像人写的。</p>
<ul>
<li><strong>什么时候用：</strong> 已有摘要或草稿，但读着像机器；准备发公众号 / 邮件 / 对客文档。</li>
<li><strong>你会得到：</strong> 同一内容的自然表述版，结构通常保留，句子更顺。</li>
<li><strong>注意：</strong> 它主要改表达，不负责重新做事实核查（核查用事实简报）。</li>
</ul>
<h3>5. 风险雷达（Risk Radar）</h3>
<p>从资讯里抓<strong>风险信号</strong>：地缘、监管合规、供应链、舆情、系统性冲击等，并尽量标出严重程度与触发条件。</p>
<ul>
<li><strong>什么时候用：</strong> 做风控扫视、出海合规关注、不想漏「坏消息」。</li>
<li><strong>你会得到：</strong> 风险清单、可能影响谁、需要盯什么信号。</li>
<li><strong>注意：</strong> 是辅助扫描，不是法律意见或投资建议。</li>
</ul>
<h3>6. 行业雷达（Sector Radar）</h3>
<p>按「赛道」看问题：供需、竞争格局、主要玩家、份额与产品/渠道壁垒。</p>
<ul>
<li><strong>什么时候用：</strong> 研究某个行业、对比竞品、写行业一页纸。</li>
<li><strong>你会得到：</strong> 谁在场上、各自位置、机会与威胁。</li>
<li><strong>注意：</strong> 上下文里行业文章越集中，结论越靠谱。</li>
</ul>
<h3>7. 宏观视图（Macro View）</h3>
<p>用宏观框架读新闻：增长、通胀、就业、流动性、央行与财政政策，以及它们对行业/资产的含义。</p>
<ul>
<li><strong>什么时候用：</strong> 宏观周报、配置背景、解释「为什么市场在动」。</li>
<li><strong>你会得到：</strong> 主叙事、关键变量、对下游含义的提示。</li>
<li><strong>注意：</strong> 需要上下文里有宏观相关稿件；纯公司八卦稿效果一般。</li>
</ul>
<h3>8. 外汇量化（Forex Quant）</h3>
<p>从利率、利差、风险偏好、宏观数据等角度，整理与汇率相关的逻辑（偏研究备忘，不是下单信号）。</p>
<ul>
<li><strong>什么时候用：</strong> 读汇市、利率决议、跨境资金相关新闻。</li>
<li><strong>你会得到：</strong> 主要驱动因素、情绪方向、风险提示。</li>
<li><strong>注意：</strong> 不替代交易系统；勿当作「保证涨跌」结论。</li>
</ul>
<h3>9. 战略官视角（CSO Board）</h3>
<p>站在公司战略层看：商业模式怎么赚钱、护城河在哪、价值如何被捕获、战略选项有哪些。</p>
<ul>
<li><strong>什么时候用：</strong> 公司深度、财报季战略解读、竞品战略对比。</li>
<li><strong>你会得到：</strong> 模式拆解、壁垒判断、可选战略路径。</li>
<li><strong>注意：</strong> 更适合有公司/商业模式信息的文章集合。</li>
</ul>
<h3>10. 加密情报（Crypto Intel）</h3>
<p>面向加密与 Web3：链上动态、DeFi 机制、代币经济、生态事件与常见风险点。</p>
<ul>
<li><strong>什么时候用：</strong> Crypto 日报、协议研究、监管对加密的影响。</li>
<li><strong>你会得到：</strong> 事件要点、机制说明、风险点列表。</li>
<li><strong>注意：</strong> 波动与骗局多，输出需结合引文回看原文。</li>
</ul>
<h3>11. 事实简报（Fact Brief）</h3>
<p>专门处理「说法互相打架」：把多源信息拆成<strong>已较一致 / 仍矛盾 / 尚未证实</strong>，减少情绪化转述。</p>
<ul>
<li><strong>什么时候用：</strong> 争议事件、传闻与辟谣并存、需要冷静事实底稿。</li>
<li><strong>你会得到：</strong> 分栏式事实清单，而不是站队长文。</li>
<li><strong>注意：</strong> 以你订阅源为准；源本身偏差大时，结论也会受限。</li>
</ul>
<h3>12. 动态分析（Dynamics）</h3>
<p>不满足于「发生了什么」，而追问<strong>变量之间怎么互相影响</strong>（因果链、正负反馈）。</p>
<ul>
<li><strong>什么时候用：</strong> 政策—市场联动、行业连锁反应、复杂系统事件。</li>
<li><strong>你会得到：</strong> 因果说明、反馈环描述、关键节点。</li>
<li><strong>注意：</strong> 比摘要更「推理向」，上下文不足时会写清不确定处（质量依赖模型与素材）。</li>
</ul>
<h3>13. 推特长文（Threader）</h3>
<p>把较长研究/综述压成适合 <strong>X / Twitter</strong> 的连续短帖（thread）：有钩子、有分条、有收尾。</p>
<ul>
<li><strong>什么时候用：</strong> 研究结论要发社交、需要传播向改写。</li>
<li><strong>你会得到：</strong> 1/ n、2/ n… 结构的帖文草稿。</li>
<li><strong>注意：</strong> 为传播会简化表述，关键数字建议用引文再核对。</li>
</ul>
<h3>14. 市场简报（Market Brief）</h3>
<p>整理成「可读的市场/行业简报」：上下游、逻辑链路、该盯的指标，结构接近投研备忘。</p>
<ul>
<li><strong>什么时候用：</strong> 早报、周度市场备忘、给团队同步。</li>
<li><strong>你会得到：</strong> 分点简报、逻辑流、关注指标。</li>
<li><strong>注意：</strong> 和「行业雷达」比，更偏成稿结构；行业雷达更偏格局拆解。</li>
</ul>
<h3>15. 科技日报（Tech Daily）</h3>
<p>聚焦科技与平台：大厂动向、云与 AI 基建、资本开支、平台壁垒与商业化进展。</p>
<ul>
<li><strong>什么时候用：</strong> 科技线日报、大厂追踪、AI 基础设施相关阅读。</li>
<li><strong>你会得到：</strong> 要点列表 + 对商业/竞争含义的短评。</li>
<li><strong>注意：</strong> 适合科技订阅较多的上下文。</li>
</ul>
<h3>16. 去噪分析（De-noiser）</h3>
<p>用尽量「第一性原理」的方式去掉废话、公关包装和正确的废话，抽出<strong>核心命题、关键数字、可执行结论</strong>。</p>
<ul>
<li><strong>什么时候用：</strong> 论文腔、超长稿、PR 稿、信息密度低但必须读的材料。</li>
<li><strong>你会得到：</strong> 极简核心结论 + 关键参数。</li>
<li><strong>注意：</strong> 会大量删细节；需要细节时再对原文展开。</li>
</ul>
<h3>17. 技术博主（Tech Onboarding）</h3>
<p>从工程实践角度评估新技术/工具/框架：学习成本、迁移摩擦、适用边界、演进风险。</p>
<ul>
<li><strong>什么时候用：</strong> 技术选型、是否引入某开源组件、读技术公告。</li>
<li><strong>你会得到：</strong> 上手成本评估、迁移注意点、适不适合你场景的判断框架。</li>
<li><strong>注意：</strong> 上下文最好包含官方文档或深度技术文，而不仅是营销通稿。</li>
</ul>
<h3>18. 思维教练（Mind Coach）</h3>
<p>针对「知道但做不到」：阅读拖延、决策瘫痪、信息过载，给出<strong>低成本下一步动作</strong>（微实验）。</p>
<ul>
<li><strong>什么时候用：</strong> 信息太多无从下手、执行卡住、需要行动清单而不是再多分析。</li>
<li><strong>你会得到：</strong> 问题诊断 + 今天就能做的一小步。</li>
<li><strong>注意：</strong> 偏个人效能，不是心理诊疗。</li>
</ul>
<h3>19. 多米诺模拟（Domino Sim）</h3>
<p>假设某个冲击已经发生，推演它在供应链/产业链上的<strong>一阶、二阶、三阶</strong>连锁影响。</p>
<ul>
<li><strong>什么时候用：</strong> 断供、制裁、天灾、区域冲突外溢、关键节点停摆。</li>
<li><strong>你会得到：</strong> 分层影响清单（直接相关方 → 间接方 → 更远端）。</li>
<li><strong>注意：</strong> 是情景推演，不是预言；假设条件应在输出里写清。</li>
</ul>
<h3>20. 林迪过滤（Lindy Filter）</h3>
<p>专门过滤「只有短热度」的噪音，保留更可能长期仍有价值的概念、原则与框架。</p>
<ul>
<li><strong>什么时候用：</strong> 热点太多想沉淀知识、做长期笔记、去 FOMO。</li>
<li><strong>你会得到：</strong> 值得保留的原则列表，以及可丢弃的短噪。</li>
<li><strong>注意：</strong> 会主动「不追新」；若你要的就是热点运营，请用趋势筛选器。</li>
</ul>
<h3>21. Serenity 方法（Serenity Method）</h3>
<p>审计系统里的<strong>关键依赖与瓶颈</strong>：物料/BOM、卡脖子环节、单点故障，以及可缓释方向。</p>
<ul>
<li><strong>什么时候用：</strong> 制造、半导体、硬件、复杂供应链、关键零部件新闻。</li>
<li><strong>你会得到：</strong> 依赖清单、瓶颈节点、可能的备选/缓释思路。</li>
<li><strong>注意：</strong> 名称来自方法论风格的分析框架；重点在瓶颈，不在鸡汤。</li>
</ul>
<h3>22. 贝叶斯交易（Bayesian Trade）</h3>
<p>用「情景 + 概率思维」讨论不确定事件：几种情形、各自大致含义、若判断错如何对冲（研究向）。</p>
<ul>
<li><strong>什么时候用：</strong> 重大事件前后、路径高度不确定、需要决策备忘而非单点预测。</li>
<li><strong>你会得到：</strong> 情景表、粗略概率讨论、对冲/应对思路。</li>
<li><strong>注意：</strong> <strong>不是</strong>荐股或下单指令；概率为示意，取决于素材与模型。</li>
</ul>

<p>若你在<strong>定时任务</strong>里配置了串行 / 并行工作流，也可用自定义指令把类似流水线固化（详见定时任务相关帮助）。</p>
<h3>3）自定义与变量</h3>
<ul>
<li>设置里可增删自己的指令，与 22 条一起显示。</li>
<li>模板变量、字段说明见：<strong>高级功能 → 快捷指令</strong>。</li>
<li>写自定义时尽量写清：角色是谁、输入是什么、输出必须长什么样、不要写什么。</li>
</ul>
<h3>4）效果不好时检查这三件事</h3>
<ol>
<li>当前对话里是不是<strong>载入了相关文章</strong>（空上下文只会空泛发挥）。</li>
<li>是否已有 <strong>AI 摘要</strong>（没有的话先让摘要跑完）。</li>
<li>指令是否和素材匹配（例如用「外汇量化」去跑纯娱乐八卦，不会有好结果）。</li>
</ol>

<h2>Part 1 — All 22 commands (read this first)</h2>
<h3>1. Ideator</h3>
<p>Finds expandable topic clusters and writing angles from your loaded articles. <strong>Use when:</strong> you need topics for the week. <strong>Output:</strong> topic map + angles. <strong>Not:</strong> a final polished essay (use Draft Builder next).</p>
<h3>2. Trend Selector</h3>
<p>Ranks which topics look hottest / highest priority in <em>your</em> feeds. <strong>Use when:</strong> too much news, need top 3–5. <strong>Output:</strong> ranked list + why. <strong>Not:</strong> a global social firehose ranking.</p>
<h3>3. Draft Builder</h3>
<p>Turns materials into an outline or first full draft. <strong>Use when:</strong> research is enough to write. <strong>Output:</strong> structure + draft body. <strong>Tip:</strong> run Humanizer before publishing.</p>
<h3>4. Humanizer</h3>
<p>Rewrites stiff/AI-sounding text into natural prose. <strong>Use when:</strong> draft exists but reads robotic. <strong>Output:</strong> same ideas, better voice. <strong>Not:</strong> fact-checking (use Fact Brief).</p>
<h3>5. Risk Radar</h3>
<p>Surfaces geopolitical, regulatory, supply-chain, and systemic risk signals. <strong>Use when:</strong> risk scan / compliance awareness. <strong>Output:</strong> risk list + triggers. <strong>Not:</strong> legal advice.</p>
<h3>6. Sector Radar</h3>
<p>Maps supply/demand, competitors, share, and moats for an industry. <strong>Use when:</strong> sector one-pagers. <strong>Output:</strong> landscape + threats/opportunities.</p>
<h3>7. Macro View</h3>
<p>Reads news through growth, inflation, liquidity, and policy. <strong>Use when:</strong> macro weekly context. <strong>Output:</strong> narrative + key variables.</p>
<h3>8. Forex Quant</h3>
<p>Organizes rates, differentials, and sentiment relevant to FX. <strong>Use when:</strong> FX / rates news. <strong>Output:</strong> drivers + caveats. <strong>Not:</strong> trade signals.</p>
<h3>9. CSO Board</h3>
<p>Strategy lens: business model, moat, value capture, options. <strong>Use when:</strong> company deep dives. <strong>Output:</strong> model breakdown + strategic options.</p>
<h3>10. Crypto Intel</h3>
<p>On-chain, DeFi, tokenomics, ecosystem events and risks. <strong>Use when:</strong> crypto daily / protocol research. <strong>Output:</strong> events + mechanisms + risks.</p>
<h3>11. Fact Brief</h3>
<p>Splits contested claims into agreed / disputed / unconfirmed. <strong>Use when:</strong> conflicting sources. <strong>Output:</strong> calm fact table, not a hot take.</p>
<h3>12. Dynamics</h3>
<p>Causal chains and feedback loops across variables. <strong>Use when:</strong> policy–market linkage or multi-step industry effects.</p>
<h3>13. Threader</h3>
<p>Compresses long analysis into X/Twitter-style threads. <strong>Use when:</strong> distribution. <strong>Tip:</strong> re-check numbers via citations.</p>
<h3>14. Market Brief</h3>
<p>Research-memo style market/industry briefing with logic flow and metrics to watch.</p>
<h3>15. Tech Daily</h3>
<p>Big-tech, platforms, CapEx, AI infra, commercialization notes.</p>
<h3>16. De-noiser</h3>
<p>Strips fluff/PR; keeps core claims and key numbers. <strong>Use when:</strong> long or low-density documents.</p>
<h3>17. Tech Onboarding</h3>
<p>Engineering view of adoption cost, migration friction, and fit. <strong>Use when:</strong> tooling decisions.</p>
<h3>18. Mind Coach</h3>
<p>Turns overwhelm into a tiny next action. <strong>Use when:</strong> execution stuck. <strong>Not:</strong> therapy.</p>
<h3>19. Domino Sim</h3>
<p>First / second / third-order shock propagation across chains. <strong>Use when:</strong> black-swan scenarios. <strong>Label assumptions.</strong></p>
<h3>20. Lindy Filter</h3>

<h3>21. Serenity Method</h3>
<p>BOM / bottleneck / choke-point audit for complex systems and hardware chains.</p>
<h3>22. Bayesian Trade</h3>
<p>Scenario + probability framing and hedge <em>ideas</em> under uncertainty. <strong>Not:</strong> stock picks or guaranteed odds.</p>

<h2>一、全部 22 条指令介绍（请先读完这一节）</h2>
<p>下面按编号列出全部内置指令。每条都写清：<strong>它是干什么的、什么时候该点它、大概会得到什么</strong>。</p>
<h3>1. 创意构思者（Ideator）</h3>
<p>从你当前加载的多篇文章里，找出「可以继续做内容」的话题方向，把零散新闻收成几个<strong>话题簇</strong>，并给出可写的角度。</p>
<ul>
<li><strong>什么时候用：</strong> 不知道写什么、要做周选题、要给频道定主题。</li>
<li><strong>你会得到：</strong> 选题清单、话题分组、每个方向可以怎么切入的建议。</li>
<li><strong>注意：</strong> 它偏「找题」，不是直接写成终稿；成稿请再配草稿构建器。</li>
</ul>
<h3>2. 趋势筛选器（Trend Selector）</h3>
<p>在已有资讯里判断哪些话题<strong>更热、更值得优先跟</strong>，按传播潜力或时效性做排序，并简单说明理由。</p>
<ul>
<li><strong>什么时候用：</strong> 信息很多、只想抓今天该盯的 3～5 个热点。</li>
<li><strong>你会得到：</strong> 热点排序、为何重要、哪些可以先放一放。</li>
<li><strong>注意：</strong> 依据的是你订阅流里的内容，不是全网热搜榜。</li>
</ul>
<h3>3. 草稿构建器（Draft Builder）</h3>
<p>把多篇素材整理成<strong>一篇结构清楚的文章骨架或初稿</strong>（标题、小标题、段落逻辑）。</p>
<ul>
<li><strong>什么时候用：</strong> 素材已经够了，要开始写深度文、周报、综述。</li>
<li><strong>你会得到：</strong> 大纲，以及可继续改的正文草稿。</li>
<li><strong>注意：</strong> 初稿可能偏「正确但平」；对外发布前常用「人性化改写」润一遍。</li>
</ul>
<h3>4. 人性化改写（Humanizer）</h3>
<p>专门改语气：减少「作为 AI」「首先其次最后」等机械感，让文字读起来像人写的。</p>
<ul>
<li><strong>什么时候用：</strong> 已有摘要或草稿，但读着像机器；准备发公众号 / 邮件 / 对客文档。</li>
<li><strong>你会得到：</strong> 同一内容的自然表述版，结构通常保留，句子更顺。</li>
<li><strong>注意：</strong> 它主要改表达，不负责重新做事实核查（核查用事实简报）。</li>
</ul>
<h3>5. 风险雷达（Risk Radar）</h3>
<p>从资讯里抓<strong>风险信号</strong>：地缘、监管合规、供应链、舆情、系统性冲击等，并尽量标出严重程度与触发条件。</p>
<ul>
<li><strong>什么时候用：</strong> 做风控扫视、出海合规关注、不想漏「坏消息」。</li>
<li><strong>你会得到：</strong> 风险清单、可能影响谁、需要盯什么信号。</li>
<li><strong>注意：</strong> 是辅助扫描，不是法律意见或投资建议。</li>
</ul>
<h3>6. 行业雷达（Sector Radar）</h3>
<p>按「赛道」看问题：供需、竞争格局、主要玩家、份额与产品/渠道壁垒。</p>
<ul>
<li><strong>什么时候用：</strong> 研究某个行业、对比竞品、写行业一页纸。</li>
<li><strong>你会得到：</strong> 谁在场上、各自位置、机会与威胁。</li>
<li><strong>注意：</strong> 上下文里行业文章越集中，结论越靠谱。</li>
</ul>
<h3>7. 宏观视图（Macro View）</h3>
<p>用宏观框架读新闻：增长、通胀、就业、流动性、央行与财政政策，以及它们对行业/资产的含义。</p>
<ul>
<li><strong>什么时候用：</strong> 宏观周报、配置背景、解释「为什么市场在动」。</li>
<li><strong>你会得到：</strong> 主叙事、关键变量、对下游含义的提示。</li>
<li><strong>注意：</strong> 需要上下文里有宏观相关稿件；纯公司八卦稿效果一般。</li>
</ul>
<h3>8. 外汇量化（Forex Quant）</h3>
<p>从利率、利差、风险偏好、宏观数据等角度，整理与汇率相关的逻辑（偏研究备忘，不是下单信号）。</p>
<ul>
<li><strong>什么时候用：</strong> 读汇市、利率决议、跨境资金相关新闻。</li>
<li><strong>你会得到：</strong> 主要驱动因素、情绪方向、风险提示。</li>
<li><strong>注意：</strong> 不替代交易系统；勿当作「保证涨跌」结论。</li>
</ul>
<h3>9. 战略官视角（CSO Board）</h3>
<p>站在公司战略层看：商业模式怎么赚钱、护城河在哪、价值如何被捕获、战略选项有哪些。</p>
<ul>
<li><strong>什么时候用：</strong> 公司深度、财报季战略解读、竞品战略对比。</li>
<li><strong>你会得到：</strong> 模式拆解、壁垒判断、可选战略路径。</li>
<li><strong>注意：</strong> 更适合有公司/商业模式信息的文章集合。</li>
</ul>
<h3>10. 加密情报（Crypto Intel）</h3>
<p>面向加密与 Web3：链上动态、DeFi 机制、代币经济、生态事件与常见风险点。</p>
<ul>
<li><strong>什么时候用：</strong> Crypto 日报、协议研究、监管对加密的影响。</li>
<li><strong>你会得到：</strong> 事件要点、机制说明、风险点列表。</li>
<li><strong>注意：</strong> 波动与骗局多，输出需结合引文回看原文。</li>
</ul>
<h3>11. 事实简报（Fact Brief）</h3>
<p>专门处理「说法互相打架」：把多源信息拆成<strong>已较一致 / 仍矛盾 / 尚未证实</strong>，减少情绪化转述。</p>
<ul>
<li><strong>什么时候用：</strong> 争议事件、传闻与辟谣并存、需要冷静事实底稿。</li>
<li><strong>你会得到：</strong> 分栏式事实清单，而不是站队长文。</li>
<li><strong>注意：</strong> 以你订阅源为准；源本身偏差大时，结论也会受限。</li>
</ul>
<h3>12. 动态分析（Dynamics）</h3>
<p>不满足于「发生了什么」，而追问<strong>变量之间怎么互相影响</strong>（因果链、正负反馈）。</p>
<ul>
<li><strong>什么时候用：</strong> 政策—市场联动、行业连锁反应、复杂系统事件。</li>
<li><strong>你会得到：</strong> 因果说明、反馈环描述、关键节点。</li>
<li><strong>注意：</strong> 比摘要更「推理向」，上下文不足时会写清不确定处（质量依赖模型与素材）。</li>
</ul>
<h3>13. 推特长文（Threader）</h3>
<p>把较长研究/综述压成适合 <strong>X / Twitter</strong> 的连续短帖（thread）：有钩子、有分条、有收尾。</p>
<ul>
<li><strong>什么时候用：</strong> 研究结论要发社交、需要传播向改写。</li>
<li><strong>你会得到：</strong> 1/ n、2/ n… 结构的帖文草稿。</li>
<li><strong>注意：</strong> 为传播会简化表述，关键数字建议用引文再核对。</li>
</ul>
<h3>14. 市场简报（Market Brief）</h3>
<p>整理成「可读的市场/行业简报」：上下游、逻辑链路、该盯的指标，结构接近投研备忘。</p>
<ul>
<li><strong>什么时候用：</strong> 早报、周度市场备忘、给团队同步。</li>
<li><strong>你会得到：</strong> 分点简报、逻辑流、关注指标。</li>
<li><strong>注意：</strong> 和「行业雷达」比，更偏成稿结构；行业雷达更偏格局拆解。</li>
</ul>
<h3>15. 科技日报（Tech Daily）</h3>
<p>聚焦科技与平台：大厂动向、云与 AI 基建、资本开支、平台壁垒与商业化进展。</p>
<ul>
<li><strong>什么时候用：</strong> 科技线日报、大厂追踪、AI 基础设施相关阅读。</li>
<li><strong>你会得到：</strong> 要点列表 + 对商业/竞争含义的短评。</li>
<li><strong>注意：</strong> 适合科技订阅较多的上下文。</li>
</ul>
<h3>16. 去噪分析（De-noiser）</h3>
<p>用尽量「第一性原理」的方式去掉废话、公关包装和正确的废话，抽出<strong>核心命题、关键数字、可执行结论</strong>。</p>
<ul>
<li><strong>什么时候用：</strong> 论文腔、超长稿、PR 稿、信息密度低但必须读的材料。</li>
<li><strong>你会得到：</strong> 极简核心结论 + 关键参数。</li>
<li><strong>注意：</strong> 会大量删细节；需要细节时再对原文展开。</li>
</ul>
<h3>17. 技术博主（Tech Onboarding）</h3>
<p>从工程实践角度评估新技术/工具/框架：学习成本、迁移摩擦、适用边界、演进风险。</p>
<ul>
<li><strong>什么时候用：</strong> 技术选型、是否引入某开源组件、读技术公告。</li>
<li><strong>你会得到：</strong> 上手成本评估、迁移注意点、适不适合你场景的判断框架。</li>
<li><strong>注意：</strong> 上下文最好包含官方文档或深度技术文，而不仅是营销通稿。</li>
</ul>
<h3>18. 思维教练（Mind Coach）</h3>
<p>针对「知道但做不到」：阅读拖延、决策瘫痪、信息过载，给出<strong>低成本下一步动作</strong>（微实验）。</p>
<ul>
<li><strong>什么时候用：</strong> 信息太多无从下手、执行卡住、需要行动清单而不是再多分析。</li>
<li><strong>你会得到：</strong> 问题诊断 + 今天就能做的一小步。</li>
<li><strong>注意：</strong> 偏个人效能，不是心理诊疗。</li>
</ul>
<h3>19. 多米诺模拟（Domino Sim）</h3>
<p>假设某个冲击已经发生，推演它在供应链/产业链上的<strong>一阶、二阶、三阶</strong>连锁影响。</p>
<ul>
<li><strong>什么时候用：</strong> 断供、制裁、天灾、区域冲突外溢、关键节点停摆。</li>
<li><strong>你会得到：</strong> 分层影响清单（直接相关方 → 间接方 → 更远端）。</li>
<li><strong>注意：</strong> 是情景推演，不是预言；假设条件应在输出里写清。</li>
</ul>
<h3>20. 林迪过滤（Lindy Filter）</h3>
<p>专门过滤「只有短热度」的噪音，保留更可能长期仍有价值的概念、原则与框架。</p>
<ul>
<li><strong>什么时候用：</strong> 热点太多想沉淀知识、做长期笔记、去 FOMO。</li>
<li><strong>你会得到：</strong> 值得保留的原则列表，以及可丢弃的短噪。</li>
<li><strong>注意：</strong> 会主动「不追新」；若你要的就是热点运营，请用趋势筛选器。</li>
</ul>
<h3>21. Serenity 方法（Serenity Method）</h3>
<p>审计系统里的<strong>关键依赖与瓶颈</strong>：物料/BOM、卡脖子环节、单点故障，以及可缓释方向。</p>
<ul>
<li><strong>什么时候用：</strong> 制造、半导体、硬件、复杂供应链、关键零部件新闻。</li>
<li><strong>你会得到：</strong> 依赖清单、瓶颈节点、可能的备选/缓释思路。</li>
<li><strong>注意：</strong> 名称来自方法论风格的分析框架；重点在瓶颈，不在鸡汤。</li>
</ul>
<h3>22. 贝叶斯交易（Bayesian Trade）</h3>
<p>用「情景 + 概率思维」讨论不确定事件：几种情形、各自大致含义、若判断错如何对冲（研究向）。</p>
<ul>
<li><strong>什么时候用：</strong> 重大事件前后、路径高度不确定、需要决策备忘而非单点预测。</li>
<li><strong>你会得到：</strong> 情景表、粗略概率讨论、对冲/应对思路。</li>
<li><strong>注意：</strong> <strong>不是</strong>荐股或下单指令；概率为示意，取决于素材与模型。</li>
</ul>

<h2>Part 1 — All 22 commands (read this first)</h2>
<h3>1. Ideator</h3>
<p>Finds expandable topic clusters and writing angles from your loaded articles. <strong>Use when:</strong> you need topics for the week. <strong>Output:</strong> topic map + angles. <strong>Not:</strong> a final polished essay (use Draft Builder next).</p>
<h3>2. Trend Selector</h3>
<p>Ranks which topics look hottest / highest priority in <em>your</em> feeds. <strong>Use when:</strong> too much news, need top 3–5. <strong>Output:</strong> ranked list + why. <strong>Not:</strong> a global social firehose ranking.</p>
<h3>3. Draft Builder</h3>
<p>Turns materials into an outline or first full draft. <strong>Use when:</strong> research is enough to write. <strong>Output:</strong> structure + draft body. <strong>Tip:</strong> run Humanizer before publishing.</p>
<h3>4. Humanizer</h3>
<p>Rewrites stiff/AI-sounding text into natural prose. <strong>Use when:</strong> draft exists but reads robotic. <strong>Output:</strong> same ideas, better voice. <strong>Not:</strong> fact-checking (use Fact Brief).</p>
<h3>5. Risk Radar</h3>
<p>Surfaces geopolitical, regulatory, supply-chain, and systemic risk signals. <strong>Use when:</strong> risk scan / compliance awareness. <strong>Output:</strong> risk list + triggers. <strong>Not:</strong> legal advice.</p>
<h3>6. Sector Radar</h3>
<p>Maps supply/demand, competitors, share, and moats for an industry. <strong>Use when:</strong> sector one-pagers. <strong>Output:</strong> landscape + threats/opportunities.</p>
<h3>7. Macro View</h3>
<p>Reads news through growth, inflation, liquidity, and policy. <strong>Use when:</strong> macro weekly context. <strong>Output:</strong> narrative + key variables.</p>
<h3>8. Forex Quant</h3>
<p>Organizes rates, differentials, and sentiment relevant to FX. <strong>Use when:</strong> FX / rates news. <strong>Output:</strong> drivers + caveats. <strong>Not:</strong> trade signals.</p>
<h3>9. CSO Board</h3>
<p>Strategy lens: business model, moat, value capture, options. <strong>Use when:</strong> company deep dives. <strong>Output:</strong> model breakdown + strategic options.</p>
<h3>10. Crypto Intel</h3>
<p>On-chain, DeFi, tokenomics, ecosystem events and risks. <strong>Use when:</strong> crypto daily / protocol research. <strong>Output:</strong> events + mechanisms + risks.</p>
<h3>11. Fact Brief</h3>
<p>Splits contested claims into agreed / disputed / unconfirmed. <strong>Use when:</strong> conflicting sources. <strong>Output:</strong> calm fact table, not a hot take.</p>
<h3>12. Dynamics</h3>
<p>Causal chains and feedback loops across variables. <strong>Use when:</strong> policy–market linkage or multi-step industry effects.</p>
<h3>13. Threader</h3>
<p>Compresses long analysis into X/Twitter-style threads. <strong>Use when:</strong> distribution. <strong>Tip:</strong> re-check numbers via citations.</p>
<h3>14. Market Brief</h3>
<p>Research-memo style market/industry briefing with logic flow and metrics to watch.</p>
<h3>15. Tech Daily</h3>
<p>Big-tech, platforms, CapEx, AI infra, commercialization notes.</p>
<h3>16. De-noiser</h3>
<p>Strips fluff/PR; keeps core claims and key numbers. <strong>Use when:</strong> long or low-density documents.</p>
<h3>17. Tech Onboarding</h3>
<p>Engineering view of adoption cost, migration friction, and fit. <strong>Use when:</strong> tooling decisions.</p>
<h3>18. Mind Coach</h3>
<p>Turns overwhelm into a tiny next action. <strong>Use when:</strong> execution stuck. <strong>Not:</strong> therapy.</p>
<h3>19. Domino Sim</h3>
<p>First / second / third-order shock propagation across chains. <strong>Use when:</strong> black-swan scenarios. <strong>Label assumptions.</strong></p>
<h3>20. Lindy Filter</h3>
<p>Keeps longer-lived ideas; drops short-lived noise. <strong>Use when:</strong> knowledge retention (not trend chasing).</p>
<h3>21. Serenity Method</h3>
<p>BOM / bottleneck / choke-point audit for complex systems and hardware chains.</p>
<h3>22. Bayesian Trade</h3>
<p>Scenario + probability framing and hedge <em>ideas</em> under uncertainty. <strong>Not:</strong> stock picks or guaranteed odds.</p>
