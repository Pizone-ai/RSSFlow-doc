# 标签过滤与内容筛选 / Filters & Content Selection

### 中文

创建定时任务时，用内容筛选缩小输入，减少噪音并控制 Token。

## 可配置维度

| 维度 | 说明 |
|------|------|
| 时间范围 | 如最近 24h / 48h / 7 天 |
| 指定订阅源 | 绑定部分源，或处理全部 |
| 标签过滤 | 按 AI 分类标签（可多选，OR） |
| 关键词 | 摘要 / 要点 / 关键词中匹配（可多词，OR） |
| 最低 AI 评分 | 如只取 ≥ 80 的文章 |
| 数量上限 | 单次最多处理篇数（默认约 100） |

## 过滤逻辑（简版）

1. 在时间窗口内取文章  
2. 默认优先已有 AI 摘要的文章（保证简报质量）  
3. 匹配订阅源与最低评分  
4. 标签 / 关键词按「满足任一即可」匹配  

## 高级搜索

支持用 `AND` / `OR` / `NOT` 组合条件，例如：

- `AI 芯片` 或 `AI AND 芯片`：同时包含  
- `智能手机 NOT 苹果`：排除词  

异常或超时会自动降级为标准检索，避免任务中断。

### English

Use content filters in the workflow editor to target high-value articles and control token cost.

## Dimensions

Time range, feeds, AI tags (OR), keywords (OR), minimum AI score, and max articles per run (~100 default).

## Flow

Window → prefer AI-summarized items → feed + score gates → tag/keyword OR matching.

## Advanced Search

Combine with `AND` / `OR` / `NOT`. On failure or timeout, the system falls back to standard search so workflows keep running.
