Центр помощи RSSFlow Reader

Здесь собраны 18 наиболее часто встречающихся вопросов пользователей с подробными ответами, которые помогут вам быстро решить возникающие при использовании вопросы.

❓ Часто задаваемые вопросы

1. Как настроить ИИ и где найти соответствующую модель?

+

Настройка ИИ выполняется в Настройки → Настройки ИИ → Включить функцию резюме ИИ. После включения переключателя настройте соответствующие параметры в конфигурации модели. В настоящее время поддерживаются четыре типа:

  1. Формат, совместимый с OpenAI: Поддерживает OpenAI и LLM, совместимые с форматом OpenAI. Введите соответствующий ключ, хост API и имя модели. Если вы используете прокси (например, gemini в OpenAI), также выберите этот тип.
    Если у вас нет аккаунта, посетите официальный сайт OpenAI https://openai.com/ для регистрации. После успешной регистрации войдите в OpenAI, перейдите на страницу управления ключами API и нажмите "Create new secret key" для создания нового ключа API.
    Кроме того, подобные модели, такие как DeepSeek, kimi, которые поддерживают формат OpenAI, также могут быть настроены с этой опцией.
  2. DeepSeek: Войдите на открытую платформу DeepSeek для создания ключа. Если у вас нет аккаунта, зарегистрируйтесь на официальном сайте: https://platform.deepseek.com/
  3. Google Gemini: Рекомендуемый выбор. Для базовой модели генерации резюме выберите gemini-1.5-flash-8b или выше для соответствия требованиям, затем настройте продвинутый LLM в пользовательских настройках для модели в режиме чата.
    Если у вас нет аккаунта, перейдите напрямую на официальный сайт Google для регистрации. Адрес: https://aistudio.google.com/app/apikey После регистрации создайте ключ API.
  4. SiliconFlow: Напрямую введите ключ API. Если в выпадающих опциях нет используемой вами модели, введите имя модели вручную. Если у вас нет аккаунта, зарегистрируйтесь напрямую. Адрес регистрации: https://account.siliconflow.cn/

2. Требуют ли режим чата и голосовой подкаст коды активации для включения? Где можно получить коды активации?

+

Режим чата и функции подкаста поддерживают 5-дневный пробный период. Коды активации предоставляются бесплатно.

Способы получения:

  • Оставьте положительный отзыв и попросите код активации у автора
  • Поделитесь или порекомендуйте расширение для получения кода активации от автора
  • Просто попросите напрямую

Важные замечания:

  • Коды активации можно использовать только один раз и они привязаны к отпечатку устройства браузера
  • Не удаляйте расширение или браузер после активации, иначе оно станет недействительным
  • После удаления необходимо использовать новый код активации для повторной активации
  • Бесплатные коды активации доступны всем, кто может видеть этот контент. Эта активность будет отменена при увеличении количества пользователей

3. ИИ успешно настроен, но DayFlow и TagFlow все еще просят включить ИИ

+

После успешной настройки ИИ, пока есть данные в резюме ИИ, данные будут отображаться в представлении DayFlow или TagFlow. В настоящее время есть два способа запустить генерацию резюме ИИ:

  1. Вы можете запустить генерацию резюме ИИ, добавив вручную в Отмеченные
  2. Вы можете включить функцию автоматического резюме в источниках подписки. Как только в источнике подписки появится новый контент, будет запущена операция автоматической генерации резюме ИИ

Предложение: После успешной настройки ИИ вы можете вручную добавить элемент в Отмеченные для проверки и тестирования правильности настройки.

Еще одна ситуация: в общих настройках, если диапазон дат запроса установлен слишком коротким, исторические данные резюме ИИ, превышающие этот диапазон дат, не будут отображаться в представлении тегов Flow.

4. Почему не удается найти входы в режим чата и функцию голосового подкаста?

+

Режим чата и голосовой подкаст основаны на ИИ и данных резюме ИИ. Когда вы успешно настроили ИИ и успешно сгенерировали резюме ИИ, вы увидите соответствующие входы для чата и подкаста в представлении Flow.

Решение: После успешной настройки ИИ вы можете вручную добавить элемент в Отмеченные. После успешной генерации эти два входа станут видимыми.

5. Как пометить как прочитанное? Можно ли найти контент, помеченный как прочитанный?

+

Метод пометки как прочитанное: Дважды щелкните левой кнопкой мыши, чтобы пометить как прочитанное. Основная операция чтения всего расширения - это двойной щелчок левой кнопкой мыши.

Особенность RSSFlow: Двойной щелчок для пометки как прочитанное, удаление карточки с воспроизведением звука деревянной рыбы - это одна из особенностей RSSFlow.

Двойной щелчок для пометки как прочитанное работает как в режиме заголовков, так и в полном чтении (иммерсивное чтение).

О прочитанном контенте: Контент, помеченный как прочитанный, больше не может быть найден, в настоящее время отображаются только непрочитанные статьи. Если статья отличная, вы можете добавить ее в избранное. Статьи в избранном всегда останутся и не будут ограничены продолжительностью хранения контента RSS и диапазоном дат запроса.

⚡ Продвинутые функции

6. В каких сценариях можно использовать горячие клавиши?

+

Горячие клавиши применяются только к полнотекстовому чтению (иммерсивное чтение). Иммерсивное чтение также является особенностью RSSFlow.

N - Следующая статья B - Предыдущая статья M - Добавить в отмеченные V - Пометить как прочитанное Enter - Открыть ссылку статьи Пробел - Страница вниз

С помощью вышеуказанных горячих клавиш вы можете полностью завершить полнотекстовое чтение через операции клавиатуры.

7. Почему некоторые новые статьи не генерируют резюме, хотя автоматическое резюме включено?

+

Только статьи с более чем 200 словами запускают генерацию резюме. Если статьи слишком короткие, например, новости из одного предложения, нет необходимости генерировать резюме.

8. Можно ли настроить теги резюме ИИ? Многие теги не те, которые мне нужны.

+

Да, можно настроить. В Настройки → Настройки ИИ → Настройки промптов есть опция "Диапазон тегов", которая соответствует пользовательским тегам.

Метод настройки: Если вы не можете найти теги ниже, введите теги вручную, разделяя каждый тег запятыми, например:

#Экономика Китая, #Экономика США, #Технологии, #Финансы

9. Поддерживает ли RSSFlow многоязычные настройки? Например, я подписываюсь на RSS-каналы на английском языке, но резюме ИИ генерируются на русском.

+

RSSFlow в настоящее время поддерживает 12 языков и может подписываться на RSS-контент любой страны на любом языке, а затем отвечать на одном из этих 12 языков.

Без языковых барьеров: RSSFlow может анализировать и обрабатывать контент любой страны и возвращать результаты анализа на выбранном языке интерфейса. Таким образом, вы можете читать и просматривать контент из многих других стран.

10. Почему представление DayFlow показывает только Мнения, а не заголовки, резюме или ключевые моменты?

+

Если статья не имеет четких точек зрения или выводов, это по сути неоднозначная статья, которая не требует детального чтения. Просто просмотр Мнений может удовлетворить основные потребности.

Если точки зрения ясны, вы можете нажать, чтобы просмотреть подробное содержание.

11. Можно ли настроить промпты резюме для отдельных статей?

+

Да, можно настроить, но это не рекомендуется.

Почему настройка не рекомендуется: Текущие промпты были тонко настроены для адаптации к LLM низкого уровня для стабильного вывода. Чтобы обеспечить правильный анализ результатов, настройка не рекомендуется. Если вам нужна настройка, пожалуйста, свяжитесь с автором.

12. Могу ли я настроить быстрые команды в режиме чата?

+

Пользовательские быстрые команды в настоящее время не поддерживаются. При необходимости вы можете напрямую вводить свои команды в окне чата.

Гибкое использование: Ручной ввод - самый гибкий режим. Вы можете ввести соответствующий промпт и запросить обработку.

13. Почему RSSFlow иногда не имеет данных? Нужно ли держать браузер открытым для получения новых данных?

+

Да, RSSFlow - это расширение браузера, которое зависит от работы браузера. Вам нужно только держать браузер открытым, и он автоматически получит обновления и обработает данные в фоновом режиме.

Понимание: Вы можете думать об этом браузере как о локальном сервисе. Пока он не закрыт, это как иметь свой собственный сервер, работающий непрерывно, помогающий вам обрабатывать RSS-данные и генерировать резюме ИИ.

14. В чем разница между RSSFlow и агентами ИИ или рабочими процессами?

+

RSSFlow на самом деле является агентом ИИ, но этот агент ИИ полностью интегрирован в сценарии чтения RSS с высокой интеграцией, что делает его больше похожим на читалку.

Вертикальный доменный агент ИИ: Его можно понимать как вертикальный доменный агент ИИ. Из-за слишком естественной интеграции он не выглядит как агент ИИ.

Отличия от рабочих процессов:

  • Рабочие процессы требуют сложных процессов, различных поддержек внешних служб извлечения контента и сложных соединений
  • Ограниченное хранение данных, негибкие операции с данными и ограниченные вычислительные ресурсы
  • RSSFlow использует локальное хранение с эффективной производительностью запросов к базе данных
  • Пока локальное хранение и производительность устройства достаточны, может хранить массивные данные
  • Может гибко собирать, фильтровать и анализировать данные
  • Значительно превосходит рабочие процессы в плане стабильности, производительности в реальном времени и эффективности

Заключение: Любой рабочий процесс не имеет конкурентоспособности по сравнению с агентом ИИ, который интегрирован с бизнесом в вертикальном домене.

15. Вы сказали, что RSSFlow может анализировать массивные данные. Я хочу проанализировать данные за 1 год. Как мне это сделать?

+

RSSFlow основан на RSS-подписках. Пока ваши подписанные RSS-источники накопили данные за 1 год, вам нужно только:

  1. В общих настройках установить продолжительность хранения RSS-контента на 365 дней
  2. Также установить диапазон дат запроса на 365 дней
  3. Тогда вы сможете запрашивать данные за 1 год

Соображения:

  • Хотя производительность запросов данных браузера отличная, не рекомендуется обрабатывать так много данных
  • Во-первых, LLM должен поддерживать такой большой контекст
  • Во-вторых, это увеличит время обработки данных, вызывая задержку браузера и более длительное время ожидания

16. В режиме чата из-за длины контекста токены потребляются очень быстро. Есть ли способ избежать этого?

+

Метод заключается в упрощении контекста через функции фильтрации данных, основанные на указанных тегах и ключевых словах для завершения выбора и фильтрации данных.

Преимущества: Это может значительно уменьшить контекст, улучшая точность анализа.

17. Почему быстрые команды иногда не работают хорошо в режиме чата?

+

Эффективность быстрых команд зависит от данных, которые вы хотите анализировать, и LLM:

  1. Релевантность данных: Данные, которые вы анализируете, должны соответствовать вашему командному сценарию. Например, если вы выполняете команду технологических новостей, но ваш контент не имеет технологического содержания, результаты определенно будут плохими.
  2. Способность LLM: Быстрые команды описывают некоторые методы анализа и бизнеса, требующие сильных способностей рассуждения или логики. Поэтому чем сильнее LLM, тем лучше результаты.
  3. Настройка параметров: Параметры LLM также влияют на конечные результаты. Вы можете соответствующим образом настроить температуру и Top P, делая некоторые тонкие настройки в написании контента или областях, требующих дивергентного мышления, для достижения оптимальных результатов.

18. У меня есть команда, которая требует особенно мощного LLM для демонстрации своей силы. Могу ли я настроить ее отдельно?

+

В настоящее время поддерживается. В режиме чата есть функция настройки ИИ, которую вы можете настроить напрямую.

Гибкая настройка: Таким образом, вы можете дифференцировать - использовать модели базовых возможностей для резюме ИИ базового контента и продвинутые модели для высокоуровневого анализа.

🔧 Устранение неполадок

🔍 Диагностика общих проблем

Не удается добавить RSS-каналы или обновление не удалось

+

Возможные причины:

  • Неправильный формат URL RSS-канала или недействительный
  • Веб-сайт не поддерживает формат RSS/Atom
  • Проблемы с сетевым подключением
  • Настройки брандмауэра или прокси блокируют доступ

Шаги решения:

  1. Проверить правильность URL RSS-канала, попробовать получить прямой доступ в браузере
  2. Подтвердить, предоставляет ли веб-сайт функциональность подписки RSS/Atom
  3. Попробовать обновить вручную или повторно добавить RSS-канал
  4. Проверить состояние сетевого подключения
  5. Временно отключить брандмауэр или прокси для тестирования

Совет: Некоторые веб-сайты могут ограничивать частоту доступа к RSS. Рассмотрите возможность соответствующей настройки интервалов обновления.

Функции ИИ не работают или генерация не удалась

+

Возможные причины:

  • Неправильная настройка ключа API
  • Недостаточный баланс аккаунта API
  • Неподходящий выбор модели
  • Проблемы с сетевым подключением
  • Служба API временно недоступна

Шаги решения:

  1. Проверить правильность настройки ключа API, повторно ввести для подтверждения
  2. Войти на веб-сайт поставщика услуг ИИ для подтверждения баланса аккаунта
  3. Попробовать переключиться на другие модели ИИ
  4. Проверить сетевое подключение, попробовать обновить страницу
  5. Проверить консоль браузера на наличие сообщений об ошибках
  6. Связаться с поставщиком услуг ИИ для подтверждения состояния службы

Примечание: Убедитесь, что выбранная модель поддерживает ваш случай использования. Рекомендуются GPT-3.5 или более высокие версии.

Расширение не может загрузиться правильно или отображается неправильно

+

Возможные причины:

  • Проблемы с кэшем браузера
  • Расширение случайно отключено
  • Версия браузера слишком старая
  • Конфликты с другими расширениями
  • Поврежденные файлы расширения

Шаги решения:

  1. Обновить страницу браузера (Ctrl+F5 для принудительного обновления)
  2. Перезапустить браузер
  3. Проверить страницу управления расширениями, подтвердить включение расширения
  4. Обновить браузер до последней версии
  5. Временно отключить другие расширения для тестирования
  6. Если проблема сохраняется, попробовать переустановить расширение

Не удается найти входы в режим чата и функцию подкаста

+

Возможные причины:

  • ИИ не настроен правильно
  • Резюме ИИ не сгенерированы
  • Код активации не введен или истек
  • Функции скрыты или отключены

Шаги решения:

  1. Подтвердить правильную настройку и нормальную работу ИИ
  2. Вручную добавить статью в Отмеченные для генерации резюме ИИ
  3. Проверить правильность ввода кода активации
  4. Искать входы чата и подкаста в представлении Flow
  5. Если все еще не найдено, пожалуйста, обратитесь в техническую поддержку

Потеря данных или проблемы синхронизации

+

Возможные причины:

  • Данные браузера были очищены
  • Расширение было переустановлено
  • Поврежденный профиль браузера
  • Недостаточно места для хранения

Важное напоминание: Данные RSSFlow хранятся локально и не синхронизируются автоматически в облако.

Меры предотвращения:

  1. Регулярно экспортировать резервные копии данных в настройках
  2. Создавать резервные копии данных перед переустановкой расширения или браузера
  3. Избегать удаления данных расширения при очистке данных браузера
  4. Обеспечить достаточное локальное пространство для хранения

⚡ Предложения по оптимизации производительности

📊 Управление хранилищем

  • Регулярно очищать просроченные статьи
  • Устанавливать разумный период хранения статей (рекомендуется 30-90 дней)
  • Своевременно удалять ненужные закладки
  • Мониторить использование локального хранилища

🔧 Оптимизация функций

  • Избегать добавления слишком многих RSS-источников (рекомендуется менее 50)
  • Отключать ненужные автоматические резюме ИИ
  • Устанавливать разумную частоту обновления RSS
  • Использовать теги и фильтрацию для лучшей эффективности

🚀 Советы по использованию

  • Использовать горячие клавиши для повышения эффективности
