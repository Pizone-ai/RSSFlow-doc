# 模型提供商配置 / Provider Configuration

### 中文

为了能使用 RSSFlow Pro 的 AI 相关服务（如摘要提炼、热点探索、智能对话等），您需要在此处配置您所拥有的大模型服务提供商。

**配置入口：** 设置页面 → 侧边栏的“AI 设置”标签页。

**启用开关：** 
- 页面顶部包含“启用 AI 功能”的总开关。只有在开启时，下方所有的 AI 配置和功能才可供使用。

**两个独立的 AI 配置选项：**
您可以根据任务的重要程度选择不同的模型以节省您的 API 流量支出：
1. **默认模型 (Basic)**：用于日常轻量化、高频的处理任务（如新文章的自动摘要生成、基本标签提取和文章初步打分评分）。建议配置体积小、速度快且性价比高的模型（如 DeepSeek-Chat 等）。
2. **复杂模型 (Advanced)**：用于深入对话、多视角研报合成、热点趋势总结等重度分析任务。建议配置推理能力极强的大参数模型（如 Claude 3.5 Sonnet、GPT-4o、Gemini 1.5 Pro 等）。

**支持的提供商：**
点击提供商图标即可直接开始配置。系统目前深度集成了以下大模型：
- **DeepSeek**：提供极具性价比的高质量智能推理与聊天。
- **OpenAI**：支持行业标准的 GPT 系列模型。
- **Anthropic Claude**：提供文笔极其流畅自然、逻辑严密的深度研报分析能力。
- **Google Gemini**：支持处理海量上下文信息的模型。
- **OpenAI Compatible 兼容服务**：允许您配置任意兼容 OpenAI 标准接口的第三方聚合商平台（如 SiliconFlow 硅基流动、火山引擎等）。
- **本地 LLM (Local)**：允许您直接连上运行在您自己电脑本地的开源模型（如 Ollama 或 LM Studio）。

**提供商参数配置说明：**
- **API 密钥 (API Key)**：您从提供商官网获取的身份令牌，出于安全考虑，输入框中的字符会自动隐藏。
- **API 接口地址 (API Host)**：提供商的官方服务器地址，系统已为您预填好了官方默认地址。如果您使用的是第三方中转或专属通道，可以自行更改。
- **模型名称 (Model Name)**：您希望运行的具体模型代号。您可以手动输入，也可以点击输入框右侧的“刷新”按钮来自动获取该服务商支持的模型列表，并直接在下拉列表中选择。

**验证与生效：**
1. **测试并暂存**：填写完参数后，点击“验证并暂存配置”按钮，系统会在后台发送一次模拟测试请求以确认配置能否连通，并通过右上角的提示反馈结果。
2. **设为启用**：测试通过后，点击“设为当前启用供应商”按钮，当前配置便会立即生效。

**关于本地运行本地模型（Ollama/LM Studio）：**
如果您选择本地 LLM，系统会非常贴心地在输入框下方展示“本地跨域访问配置指南”：
- **桌面版 Ollama**：需要在系统环境变量中配置允许浏览器扩展访问。
- **命令行 Ollama**：需要在启动命令中附带 `OLLAMA_ORIGINS="chrome-extension://*"` 参数。
- **LM Studio**：直接在客户端设置中勾选允许跨域访问即可。

### English

To use the automated features of RSSFlow Pro (such as summary generation, topic discovery, and AI chat), you need to configure your AI model providers.

**Access:** Settings page → "AI Settings" tab in the sidebar.

**Master Switch:**
- Located at the top of the settings page. When toggled off, all AI features and configurations will be hidden and suspended.

**Two Profiles:**
You can set up two different AI configurations to balance processing costs and analytics quality:
- **Base Model (Basic)**: Used for daily light tasks (e.g., auto-summarization, basic tagging, and scoring of new incoming articles). We recommend faster, cost-effective models (such as `deepseek-chat`).
- **Advanced Model (Advanced)**: Used for deep discussions, multi-perspective synthesis, and trend clustering. We recommend high-performance models (such as `claude-3-5-sonnet`, `gpt-4o`, or `gemini-1.5-pro`).

**Supported Providers:**
Click any provider icon in the grid to configure it:
- **DeepSeek**: Known for high-quality reasoning with industry-leading affordability.
- **OpenAI**: Supports standard GPT series models.
- **Anthropic**: Excellent for generating highly articulate and logical research briefs.
- **Google Gemini**: Ideal for processing large batches of text due to its massive context window.
- **OpenAI Compatible**: Allows you to connect custom endpoints or aggregator platforms (e.g., SiliconFlow, OpenRouter) matching standard OpenAI syntax.
- **Local LLM**: Connects directly to open-source models running on your local machine via Ollama or LM Studio.

**Configuration Parameters:**
- **API Key**: The authentication token obtained from your provider's developer console. Characters are masked for security.
- **API Host**: The server endpoint. Official endpoints are pre-filled by default. If you use a custom proxy or dedicated endpoint, you can modify it here.
- **Model Name**: The exact name code of the model you wish to use. You can type it manually, or click the refresh button on the right to discover and select from available models.

**Validation & Activation:**
1. **Validate & Stash**: Click "Validate & Save Config" to test the connection in the background. A notification will show whether the test succeeded.
2. **Set Active**: Once validated, click "Set as Active Provider" to apply the settings immediately.

**Running Local Models (Ollama/LM Studio):**
If you choose Local LLM, a "Local CORS Setup Guide" will automatically display below the Host input:
- **Ollama Desktop**: Add system environment variables to allow extension authorization.
- **Ollama CLI**: Run Ollama with the `$env:OLLAMA_ORIGINS="chrome-extension://*"` prefix.
- **LM Studio**: Check the "Enable CORS" toggle in the client settings.
