Centre d'aide RSSFlow Reader

Nous avons rassemblé ici les 18 questions les plus fréquemment rencontrées par les utilisateurs avec des réponses détaillées pour vous aider à résoudre rapidement vos interrogations lors de l'utilisation.

❓ Questions fréquemment posées

1. Comment configurer l'IA et où s'inscrire pour les modèles d'IA ?

+

La configuration de l'IA se fait dans Paramètres → Paramètres IA → Activer la fonction de résumé IA. Après avoir activé l'interrupteur, configurez les paramètres correspondants dans la configuration du modèle. Actuellement, quatre types sont pris en charge :

  1. Format compatible OpenAI : Prend en charge OpenAI et les LLM compatibles avec le format OpenAI. Saisissez la clé, l'hôte API et le nom du modèle correspondants. Si vous utilisez un proxy (comme gemini vers OpenAI), choisissez également ce type.
    Si vous n'avez pas de compte, visitez le site officiel d'OpenAI https://openai.com/ pour vous inscrire. Après une inscription réussie, connectez-vous à OpenAI, accédez à la page de gestion des clés API et cliquez sur "Create new secret key" pour créer une nouvelle clé API.
    De plus, des modèles similaires comme DeepSeek, kimi qui prennent en charge le format OpenAI peuvent également être configurés avec cette option.
  2. DeepSeek : Connectez-vous à la plateforme ouverte DeepSeek pour créer une clé. Si vous n'avez pas de compte, inscrivez-vous sur le site officiel : https://platform.deepseek.com/
  3. Google Gemini : Choix recommandé. Pour la génération de résumés, sélectionnez gemini-1.5-flash-8b ou supérieur dans le modèle de base pour répondre aux exigences, puis configurez un LLM de haut niveau dans les paramètres personnalisés pour le modèle en mode chat.
    Si vous n'avez pas de compte, inscrivez-vous directement sur le site officiel de Google. Adresse : https://aistudio.google.com/app/apikey Après inscription, créez une clé API.
  4. SiliconFlow : Saisissez directement la clé API. Si le modèle que vous utilisez n'est pas dans les options déroulantes, saisissez manuellement le nom du modèle. Si vous n'avez pas de compte, inscrivez-vous directement. Adresse d'inscription : https://account.siliconflow.cn/

2. Le mode chat et le podcast vocal nécessitent-ils un code d'activation pour être activés ? Où peut-on obtenir le code d'activation ?

+

Le mode chat et la fonction podcast prennent en charge une période d'essai de 5 jours, et les codes d'activation sont tous obtenus gratuitement.

Méthodes d'obtention :

  • Faire une bonne évaluation et demander un code d'activation à l'auteur
  • Publier un partage ou faire une recommandation pour obtenir un code d'activation de l'auteur
  • Demander directement

Notes importantes :

  • Le code d'activation ne peut être utilisé qu'une seule fois et sera lié à l'empreinte de l'appareil du navigateur
  • Après activation, ne supprimez pas l'extension ou le navigateur, sinon cela deviendra invalide
  • Après suppression, une réutilisation nécessite une réactivation avec un nouveau code d'activation
  • L'obtention gratuite du code d'activation est valide pour quiconque peut voir ce contenu. Cette activité sera annulée lorsque le nombre d'utilisateurs augmentera

3. L'IA a été configurée avec succès, mais DayFlow et TagFlow demandent toujours d'activer l'IA

+

Après une configuration IA réussie, tant qu'il y a des données dans le résumé IA, des données s'afficheront dans la vue DayFlow ou TagFlow. Actuellement, il existe deux façons de déclencher la génération de résumés IA :

  1. Vous pouvez déclencher la génération de résumés IA en ajoutant manuellement à Marked
  2. Vous pouvez activer la fonction de résumé automatique dans la source d'abonnement. Dès qu'il y a du nouveau contenu dans la source d'abonnement, l'opération de génération automatique de résumé IA sera déclenchée

Suggestion : Après une configuration IA réussie, vous pouvez ajouter manuellement un élément à Marked pour vérifier et tester si la configuration est correcte.

Une autre situation : dans les paramètres généraux, si la plage de dates de requête est définie trop courte, les données de résumé IA historiques dépassant cette plage de dates ne s'afficheront pas dans la vue des tags Flow.

4. Pourquoi ne trouve-t-on pas les entrées du mode chat et de la fonction podcast vocal ?

+

Le mode chat et le podcast vocal sont tous deux basés sur l'IA et les données de résumé IA. Lorsque vous avez configuré l'IA avec succès et qu'un résumé IA a été généré avec succès, vous verrez les entrées correspondantes pour le chat et le podcast dans la vue Flow.

Solution : Après une configuration IA réussie, vous pouvez ajouter manuellement un élément à Marked. Une fois la génération réussie, ces deux entrées seront visibles.

5. Comment marquer comme lu ? Le contenu marqué comme lu peut-il encore être trouvé ?

+

Méthode de marquage comme lu : Double-cliquez avec le bouton gauche de la souris pour marquer comme lu. L'opération de lecture principale de toute l'extension est le double-clic avec le bouton gauche de la souris.

Caractéristique de RSSFlow : Double-cliquer pour marquer comme lu, effacer la carte tout en jouant le son du poisson en bois, c'est l'une des caractéristiques de RSSFlow.

Le double-clic pour marquer comme lu fonctionne à la fois en mode titre et en lecture complète (lecture immersive).

À propos du contenu lu : Le contenu marqué comme lu ne peut plus être retrouvé, actuellement seuls les articles non lus sont affichés. Si un article est excellent, vous pouvez le mettre en favoris. Les articles mis en favoris resteront toujours et ne seront pas limités par la durée de rétention du contenu RSS et la plage de dates de requête.

⚡ Utilisation des fonctions avancées

6. Dans quels scénarios les raccourcis clavier peuvent-ils être utilisés ?

+

Les opérations par raccourcis clavier ne s'appliquent qu'à la lecture complète (lecture immersive). La lecture immersive est également une caractéristique de RSSFlow.

N - Article suivant B - Article précédent M - Ajouter aux marqués V - Marquer comme lu Entrée - Ouvrir le lien de l'article Espace - Tourner la page

Grâce aux raccourcis ci-dessus, vous pouvez terminer la lecture complète entièrement au clavier.

7. Pourquoi certains nouveaux articles, bien que la source d'abonnement ait activé la fonction de génération automatique de résumés, n'ont-ils pas réussi à générer des résumés ?

+

Seuls les articles de plus de 200 caractères déclencheront la génération de résumés. Si l'article est trop court, comme une nouvelle d'une phrase, il n'est pas nécessaire de générer un résumé.

8. Les tags des résumés IA peuvent-ils être personnalisés ? Beaucoup de tags ne sont pas ceux que je veux.

+

Ils peuvent être personnalisés. Dans Paramètres → Paramètres IA → Paramètres des invites, il y a une "Plage de tags" qui correspond aux tags personnalisés de l'utilisateur.

Méthode de configuration : Si vous ne trouvez pas de Tags en bas, saisissez manuellement les tags, chaque tag séparé par une virgule, comme ceci :

#Économie chinoise, #Économie américaine, #Technologie, #Finance

9. RSSFlow prend-il en charge les paramètres multilingues ? Par exemple, je m'abonne à des sources RSS en anglais, mais les résumés IA générés et les réponses de chat sont en français.

+

RSSFlow prend actuellement en charge 12 langues, peut s'abonner au contenu RSS de n'importe quel pays et dans n'importe quelle langue, puis répondre dans l'une de ces 12 langues nationales.

Sans barrière linguistique : RSSFlow peut analyser et traiter le contenu de n'importe quel pays et retourner les résultats d'analyse dans la langue sélectionnée de l'interface. Cela vous permet de lire et consulter le contenu de sources de nombreux autres pays.

10. Pourquoi la vue DayFlow n'affiche-t-elle que les Opinions et non les titres, résumés ou contenus liés aux points clés ?

+

Si un article n'a pas de points de vue ou de conclusions clairs, c'est en fait un article évasif qui ne nécessite pas une lecture détaillée. Regarder directement les Opinions peut satisfaire les besoins de base.

Si les points de vue sont clairs, vous pouvez cliquer pour voir le contenu détaillé.

11. Les invites de résumé pour des articles individuels peuvent-elles être personnalisées ?

+

Oui, elles peuvent être personnalisées, mais ce n'est pas recommandé.

Pourquoi la personnalisation n'est pas recommandée : Les invites actuelles ont été ajustées pour s'adapter aux LLM de bas niveau pour une sortie stable. Pour garantir que les résultats puissent être correctement analysés, la personnalisation n'est pas recommandée. Si vous avez besoin de personnalisation, veuillez contacter l'auteur.

12. Peut-on personnaliser les commandes rapides en mode chat ?

+

Les commandes rapides personnalisées ne sont actuellement pas prises en charge. Si nécessaire, vous pouvez saisir directement vos commandes dans la fenêtre de chat.

Utilisation flexible : La saisie manuelle est le mode le plus flexible. Vous pouvez saisir votre invite correspondante et demander un traitement.

13. Pourquoi RSSFlow n'a-t-il parfois pas de données ? Faut-il garder le navigateur ouvert en permanence pour obtenir de nouvelles données ?

+

Oui, RSSFlow est une extension de navigateur qui dépend du fonctionnement du navigateur. Vous devez juste garder le navigateur ouvert, et il obtiendra automatiquement les mises à jour et traitera les données en arrière-plan.

Compréhension : Vous pouvez considérer ce navigateur comme un service local. Tant qu'il n'est pas fermé, c'est comme avoir votre propre serveur qui fonctionne en continu, vous aidant à traiter les données RSS et à générer des résumés IA.

14. Quelle est la différence entre RSSFlow et les agents IA ou les flux de travail ?

+

RSSFlow est en fait aussi un Agent IA, mais cet Agent IA est entièrement intégré dans les scénarios de lecture RSS avec une intégration élevée, ce qui le fait ressembler davantage à un lecteur.

Agent IA de domaine vertical : Il peut être compris comme un Agent IA de domaine vertical. En raison d'une intégration trop naturelle, il ne ressemble pas à un Agent IA.

Différences avec les flux de travail :

  • Les flux de travail nécessitent des processus complexes, divers supports de services d'extraction de contenu externe et des connexions complexes
  • Le stockage de données est limité, les opérations de données ne sont pas flexibles et les ressources de calcul sont restreintes
  • RSSFlow utilise le stockage local avec des performances de requête de base de données efficaces
  • Tant que le stockage local et les performances de l'appareil sont suffisants, il peut stocker des quantités massives de données
  • Peut assembler, filtrer et analyser les données de manière flexible
  • Dépasse largement les flux de travail en termes de stabilité, de temps réel et d'efficacité

Conclusion : Tout flux de travail n'a aucune compétitivité face à un Agent IA intégré aux affaires dans un domaine vertical.

15. Vous dites que RSSFlow peut analyser des données massives. Je veux analyser 1 an de données. Comment dois-je procéder ?

+

RSSFlow est basé sur les abonnements RSS. Tant que vos sources RSS abonnées ont accumulé 1 an de données, vous devez juste :

  1. Dans les paramètres généraux, définir la durée de rétention du contenu RSS à 365 jours
  2. Définir également la plage de dates de requête à 365 jours
  3. Alors vous pourrez interroger 1 an de données

Considérations :

  • Bien que les performances de requête de données du navigateur soient excellentes, il n'est pas recommandé de traiter autant de données
  • D'abord, le LLM doit prendre en charge un contexte aussi large
  • Ensuite, cela augmentera le temps de traitement des données, causant des ralentissements du navigateur et des temps d'attente plus longs

16. En mode chat, en raison de la longueur du contexte, les tokens sont consommés très rapidement. Y a-t-il un moyen d'éviter cela ?

+

La méthode consiste à rationaliser le contexte grâce aux fonctions de filtrage des données, en se basant sur des tags spécifiés et des mots-clés pour compléter la sélection et le filtrage des données.

Avantages : Cela peut considérablement réduire le contexte tout en améliorant la précision de l'analyse.

17. Pourquoi les commandes rapides ne fonctionnent-elles parfois pas bien en mode chat ?

+

L'efficacité des commandes rapides dépend des données que vous voulez analyser et du LLM :

  1. Pertinence des données : Les données que vous analysez doivent correspondre au scénario de votre commande. Par exemple, si vous exécutez une commande de rapport technologique mais que votre contenu n'a pas de contenu lié à la technologie, les résultats seront certainement mauvais.
  2. Capacité du LLM : Les commandes rapides décrivent certaines méthodes d'analyse et d'affaires, nécessitant de fortes capacités de raisonnement ou logiques. Par conséquent, plus le LLM est puissant, meilleurs sont les résultats.
  3. Ajustement des paramètres : Les paramètres du LLM affectent également les résultats finaux. Vous pouvez ajuster appropriément la Température et le Top P, faire quelques ajustements fins dans la rédaction de contenu ou les domaines nécessitant une pensée divergente pour obtenir des résultats optimaux.

18. J'ai une commande qui nécessite un LLM particulièrement puissant pour montrer sa force. Peut-elle être configurée séparément ?

+

Actuellement pris en charge. En mode chat, il y a une fonction de paramètres IA que vous pouvez configurer directement.

Configuration flexible : Ainsi vous pouvez différencier - utiliser des modèles de capacité de base pour les résumés IA de contenu de base, et des modèles avancés pour l'analyse de haut niveau.

🔧 Dépannage

🔍 Diagnostic des problèmes courants

Impossible d'ajouter des flux RSS ou échec de mise à jour

+

Causes possibles :

  • Erreur de format d'URL du flux RSS ou invalide
  • Le site web ne prend pas en charge le format RSS/Atom
  • Problèmes de connexion réseau
  • Paramètres de pare-feu ou proxy bloquant l'accès

Étapes de résolution :

  1. Vérifier si l'URL du flux RSS est correcte, essayer d'y accéder directement dans le navigateur
  2. Confirmer si le site web fournit une fonctionnalité d'abonnement RSS/Atom
  3. Essayer de rafraîchir manuellement ou de rajouter le flux RSS
  4. Vérifier l'état de la connexion réseau
  5. Désactiver temporairement le pare-feu ou le proxy pour les tests

Conseil : Certains sites web peuvent limiter la fréquence d'accès RSS. Considérez ajuster les intervalles de mise à jour de manière appropriée.

Les fonctions IA ne fonctionnent pas ou la génération échoue

+

Causes possibles :

  • Erreur de configuration de la clé API
  • Solde insuffisant du compte API
  • Sélection de modèle inappropriée
  • Problèmes de connexion réseau
  • Service API temporairement indisponible

Étapes de résolution :

  1. Vérifier si la clé API est correctement configurée, ressaisir pour confirmer
  2. Se connecter au site web du fournisseur de services IA pour confirmer le solde du compte
  3. Essayer de passer à d'autres modèles IA
  4. Vérifier la connexion réseau, essayer de rafraîchir la page
  5. Vérifier la console du navigateur pour les messages d'erreur
  6. Contacter le fournisseur de services IA pour confirmer l'état du service

Note : Assurez-vous que le modèle sélectionné prend en charge votre cas d'usage. GPT-3.5 ou des versions supérieures sont recommandées.

L'extension ne peut pas se charger correctement ou s'affiche anormalement

+

Causes possibles :

  • Problèmes de cache du navigateur
  • Extension accidentellement désactivée
  • Version du navigateur trop ancienne
  • Conflits avec d'autres extensions
  • Fichiers d'extension corrompus

Étapes de résolution :

  1. Rafraîchir la page du navigateur (Ctrl+F5 pour un rafraîchissement forcé)
  2. Redémarrer le navigateur
  3. Vérifier la page de gestion des extensions, confirmer que l'extension est activée
  4. Mettre à jour le navigateur vers la dernière version
  5. Désactiver temporairement d'autres extensions pour les tests
  6. Si le problème persiste, essayer de réinstaller l'extension

Impossible de trouver les entrées du mode chat et de la fonction podcast

+

Causes possibles :

  • IA pas correctement configurée
  • Aucun résumé IA généré
  • Code d'activation non saisi ou expiré
  • Fonctions cachées ou désactivées

Étapes de résolution :

  1. Confirmer que l'IA est correctement configurée et fonctionne normalement
  2. Ajouter manuellement un article aux Marqués pour générer un résumé IA
  3. Vérifier si le code d'activation est correctement saisi
  4. Chercher les entrées chat et podcast dans la vue Flow
  5. Si toujours introuvable, veuillez contacter le support technique

Perte de données ou problèmes de synchronisation

+

Causes possibles :

  • Données du navigateur effacées
  • Extension réinstallée
  • Profil du navigateur corrompu
  • Espace de stockage insuffisant

Rappel important : Les données RSSFlow sont stockées localement et ne se synchronisent pas automatiquement vers le cloud.

Mesures préventives :

  1. Exporter régulièrement les sauvegardes de données dans les paramètres
  2. Sauvegarder les données avant de réinstaller l'extension ou le navigateur
  3. Éviter de supprimer les données d'extension lors du nettoyage des données du navigateur
  4. Assurer un espace de stockage local suffisant

⚡ Suggestions d'optimisation des performances

📊 Gestion du stockage

  • Nettoyer régulièrement les articles expirés
  • Définir une période de rétention d'articles raisonnable (30-90 jours recommandés)
  • Supprimer rapidement les favoris inutiles
  • Surveiller l'utilisation du stockage local

🔧 Optimisation des fonctions

  • Éviter d'ajouter trop de flux RSS (moins de 50 recommandés)
  • Désactiver les résumés IA automatiques inutiles
  • Définir une fréquence de mise à jour RSS raisonnable
  • Utiliser les tags et le filtrage pour une meilleure efficacité

🚀 Conseils d'utilisation

  • Utiliser les raccourcis clavier pour améliorer l'efficacité
  • Bien utiliser le mode de lecture immersive
  • Optimiser l'analyse IA grâce au filtrage par tags
  • Sauvegarder régulièrement les données importantes
