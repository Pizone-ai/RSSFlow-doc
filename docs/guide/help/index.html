1. 如何配置AI，去哪里找对应的模型数据？

+

配置AI在 设置 → AI设置 → AI摘要功能开启，打开开关后，在模型配置中配置对应的配置即可。当前支持以下四种类型：

  1. OpenAI 兼容格式：支持OpenAI，以及支持OpenAI格式的LLM都可以，填写对应的key和api host以及model name。如果使用代理（比如gemini转OpenAI），也选择这个类型。
  2. DeepSeek：登录DeepSeek开放平台，创建key即可。
  3. Google Gemini：推荐选择，在生成摘要的基础模型选择gemini-1.5-flash-8b或以上版本。
  4. SiliconFlow：直接填写api key，如果下拉选项没有使用的模型，则直接手动输入模型名称即可。

相关链接：

• OpenAI: https://openai.com/

• DeepSeek: https://platform.deepseek.com/

• Google Gemini: https://aistudio.google.com/app/apikey

• SiliconFlow: https://account.siliconflow.cn/

2. 聊天模式和语音播客必须要激活码才能开启吗？激活码哪里可以获得？

+

聊天模式和播客功能支持5天的试用期，激活码都是免费获得的。

获取方式：

  • 进行好评，就可以跟作者要激活码
  • 发帖分享或做推荐也可以获得作者的激活码
  • 主动索要即可

重要注意：

  • 激活码只能使用一次，会绑定浏览器设备指纹
  • 激活后请勿删除扩展或删除浏览器，否则会失效
  • 删除后再次使用需要重新使用新的激活码进行激活
  • 激活码的免费索取，只要能看到这条内容，都有效。后续用户量增加会取消这个活动

3. 配置成功AI，但是在DayFlow和TagFlow中还是提示需要去开启AI

+

AI配置成功后，只要AI摘要有数据，那么在DayFlow或TagFlow视图就会有显示数据。当前有两种方式可以触发AI生成摘要：

  1. 可以通过手动添加Marked来触发生成AI摘要
  2. 可以在订阅源中打开自动摘要功能，只要订阅源有新的内容，就会触发自动生成AI摘要的操作

建议：配置成功AI后，可以手动添加一条Marked，来完成验证测试配置是否OK。

另一种情况：在常规设置中，查询日期范围设置的太短，那么超过这个日期范围的历史AI摘要数据不会显示在Flow的标签视图中。

4. 为何没有找到聊天模式和语音播客功能的入口？

+

聊天模式和语音播客都是基于AI和AI摘要数据的，当你成功配置AI后，只要有生成成功过AI摘要，那么在Flow视图中就会看到对应的聊天和播客这两个入口了。

解决方法：AI配置成功后，可以手动添加一条数据到Marked，生成成功后，这两个入口就可以看到了。

5. 怎么才能标记已读，标记为已读的内容，还可以找到吗？

+

标记已读方法：双击鼠标左键为标记为已读，整个扩展的一个核心阅读操作就是双击鼠标左键。

RSSFlow特色：双击标记已读，消除卡片同时播放木鱼声音，这是rssflow的其中一个特色。

双击标记为已读，在标题模式和全文阅读（沉浸式阅读）都是可以生效。

关于已读内容：标记为已读的内容，无法再次找到，当前只显示未读的文章。如果文章很棒，可以收藏，收藏后的文章会一直都在，不会受RSS内容保留时长和查询日期范围的限制。

6. 快捷键设置在哪个场景下可以使用？

+

快捷键操作只适用于全文阅读（沉浸式阅读），沉浸式阅读也是rssflow一个特色。

N - 下一篇文章 B - 上一篇文章 M - 添加到marked V - 标记为已读 Enter - 打开文章链接 空格 - 翻页

通过上述快捷键，全程键盘操作来完成全文的阅读。

7. 为什么有些新的文章，订阅源也开启了自动生成摘要功能，但是没成功生成摘要？

+

只有文章超过200字的才会触发生成摘要，如果文章太短，一句话新闻，是没有必要生成摘要的。

8. AI摘要的标签可以自定义吗？很多标签并不是我想要的。

+

可以自定义，在 设置 → AI设置 → 提示词设置 中有个"标签范围"，这个标签范围就是对应用户的自定义标签。

设置方法：如果在下方没有找到Tags，则手动自己输入标签，每个标签用逗号分隔，类似这样：

#Chinese Economy, #US economy, #Technology, #Finance

9. RSSFlow支持多语言设置吗？比如我订阅英文的RSS源，但生成的AI摘要是中文。

+

RSSFlow目前支持12种语言，可以订阅任何国家的、任何语言的RSS内容，然后以这12种国家语言中的一个来回复。

无语言障碍：RSSFlow可以分析和处理任何国家的内容，以界面所选的语言进行返回分析结果。这样你就可以阅读和查看很多其它国家的源的内容。

10. 为什么DayFlow视图中，只显示Opinions，不显示标题、摘要、或者要点相关的内容？

+

一篇文章，如果没有明确的观点或者结论，其实就是一篇和稀泥的文章，没有必要细看，直接看观点Opinions就可以满足基本需求了。

如果观点明确，可以点开详细的内容查看。

11. 单篇文章的摘要提示词可以自定义吗？

+

可以进行自定义，但是不建议自定义。

不建议自定义的原因：当前的提示词做了适配低级别LLM可以稳定输出的微调，为了确保结果能正常被解析，不建议自定义。如果需要自定义，可以联系作者。

12. 在聊天模式中，可以自定义快捷指令吗？

+

暂不支持自定义快捷指令，如果有需要，直接在聊天窗口，输入你的指令即可。

灵活使用：手动输入就是最灵活的模式，你输入你对应的prompt，要求进行处理即可。

13. RSSFlow为何有时候没有数据，是否一定要一直开启浏览器，才能获取到新的数据？

+

是这样的，RSSFlow是浏览器扩展，依赖于浏览器运行，你只需要保持浏览器打开，后台就会自动获取更新和处理数据。

理解方式：你可以理解这个浏览器类似一个本地服务，只要不关闭就相当于有一台自己的服务器，在不间断的运行，帮你处理RSS数据，生成AI摘要。

14. RSSFlow跟AI Agent或工作流有什么区别？

+

RSSFlow实际上也是一个AI Agent，只不过这个AI Agent是充分融入到RSS阅读场景中，与场景进行了高度的融合，看起来更像是个阅读器。

垂直领域AI Agent：可以理解为垂直领域的AI Agent，由于融合的过于自然，导致不像是AI Agent。

与工作流的区别：

  • 工作流需要复杂的流程，各种外部抓取内容服务支持，还有各种复杂连线
  • 数据存储也有限，数据操作不灵活，运算资源受限
  • 而RSSFlow是本地存储，数据库查询性能高效
  • 只要本地存储足够和设备性能足够，可以存储海量的数据
  • 可以灵活组装和过滤和分析数据
  • 在稳定性、实时性和高效性来说，都是远超工作流的

结论：任何工作流在一个垂直领域的与业务融合在一起的AI Agent面前都是没有竞争力的。

15. 你说RSSFlow可以分析海量数据，我想要分析1年的数据，需要怎么做？

+

RSSFlow是基于RSS订阅，只要你订阅RSS源的数据积累有1年的数据，你只需要：

  1. 在常规设置中，把RSS内容保留时长设置为365天
  2. 查询日期范围也设置成365天
  3. 那么就可以查询出1年的数据

注意事项：

  • 虽然浏览器的数据查询性能很优秀，但也不建议处理这么多的数据
  • 首先LLM需要支持这么大的上下文
  • 其次会增加数据处理的时间，造成浏览器卡顿，增加等待时长

16. 聊天模式下，因为上下文长度的关系，token消耗特别快，有方法可以避免吗？

+

方法就是对上下文进行精简，通过数据筛选功能，根据指定的标签，指定的关键词，完成数据的挑选和过滤。

优势：这样可以大幅缩减上下文，同时又能提高分析的精准性。

17. 为何有时候聊天模式下，快捷指令效果不好？

+

快捷指令的效果，取决于你要分析的数据和LLM：

  1. 数据契合度：你分析的数据跟你的指令的场景需要契合，比如如果你执行科技日报的指令，但是你的内容中却没有科技相关的内容，那肯定出来的效果是不好的。
  2. LLM能力：快捷指令对一些分析和业务方法有描述，需要强大的推理或逻辑能力，因此LLM能力越强，效果越好。
  3. 参数调整：LLM的参数对最终的效果也是有影响的，可以适当调整Temperature和Top P，在内容写作或需要发散的领域做一些微调，以达到最优的效果。

18. 我有个指令需要特别强的LLM才能发挥出实力，可以进行自定义配置吗？

+

当前可以支持，在聊天模式下，有个AI设置的功能，直接配置即可。

灵活配置：这样你就可以区分开来，基础内容的AI摘要使用基础能力弱的模型，高级分析用高级模型。

🔧 故障排除

RSS源无法添加或更新失败

  • 检查RSS源URL是否正确
  • 确认网站支持RSS/Atom格式
  • 尝试刷新或重新添加
  • 检查网络连接

AI功能不工作

  • 检查API密钥是否正确配置
  • 确认API账户有足够余额
  • 检查网络连接
  • 尝试切换到其他AI模型

扩展无法正常加载

  • 刷新浏览器页面
  • 重启浏览器
  • 检查扩展是否被禁用
  • 尝试重新安装扩展

数据丢失或同步问题

  • 数据存储在本地，不会自动同步
  • 可以在设置中导出数据备份
  • 重装扩展前请先备份数据

性能优化建议

  • 定期清理过期文章
  • 合理设置文章保留期限
  • 避免添加过多RSS源
  • 关闭不需要的AI自动摘要

🚀 Installation Guide

Install from Browser Store

  1. Chrome Users: Visit Chrome Web Store
  2. Edge Users: Visit Microsoft Edge Add-ons
  3. Click "Add to Chrome" or "Get" button
  4. Confirm installation permissions
  5. After installation, the extension icon will appear in your browser toolbar

Initial Setup

  1. Click the extension icon to open the sidebar
  2. Click the settings button (gear icon)
  3. Select your preferred language
  4. Configure AI services (optional, for summary and chat features)
Tip: After first installation, we recommend adding a few RSS feeds to experience basic functionality before configuring AI features.

Quick Setup

  1. Go to Settings → AI Settings → Enable AI Summary

❓ Frequently Asked Questions

For the complete list of 18 detailed FAQ items, please switch to the Chinese version above. Here are the most important ones:

How to get activation codes for chat and podcast features?

+

Chat mode and podcast features offer a 5-day trial period. Activation codes are free and can be obtained by:

  • Leaving a positive review
  • Sharing or recommending the extension
  • Contacting the author directly

Important: Activation codes can only be used once and are bound to your browser. Don't delete the extension or browser after activation.

Why can't I see chat and podcast features?

+

These features require AI configuration and generated summaries. After configuring AI successfully, manually add an article to Marked to generate a summary, then the chat and podcast entries will appear in Flow view.

How to mark articles as read?

+

Double-click the left mouse button to mark as read. This is a core feature of RSSFlow and works in both title mode and immersive reading mode. You'll hear a wooden fish sound when marking as read.

🔧 Troubleshooting

Common Issues

  • RSS feeds not updating: Check URL, network connection, and try refreshing
  • AI features not working: Verify API key, account balance, and model selection
  • Extension not loading: Refresh page, restart browser, or reinstall extension
  • Missing chat/podcast features: Ensure AI is configured and summaries are generated

Performance Tips

  • Regularly clean up old articles
  • Set reasonable retention periods
  • Limit number of RSS feeds
  • Use data filtering for better performance
