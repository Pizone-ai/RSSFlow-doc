# 快捷指令 / AI Commands

### 中文

快捷指令（也叫**指令中心**）让你在 AI 聊天里**一键换专家角色**：不用自己写很长的提示词，点一下就能让 AI 按固定专业方式处理当前文章。

- 系统内置 **22** 条专家指令（部分需 **Pro**）。  
- 你也可以自己添加指令，和系统指令出现在**同一个菜单**里。  
- 每一条「它到底干什么、什么时候点」的完整说明，在：

> **AI 智能核心 → 专家指令集**（建议先通读「全部 22 条介绍」）

本页重点：**从哪里打开、每条指令用白话说明、如何自定义**。

## 从哪里打开

| 位置 | 怎么找 |
|------|--------|
| AI 聊天页 | 输入框附近的 **指令 / 命令** 图标，打开指令列表后点选 |
| 阅读相关侧栏 | 部分界面底部有 **指令中心** 入口 |
| 信息流分组 | 先点分组上的 AI 聊天带入该组文章，再在聊天里开指令 |
| 设置 | 打开指令管理：新建、修改、删除**你自己的**指令 |

**第一次用请记住三步：**

1. 先让对话里**有文章上下文**（不要空聊）。  
2. 打开指令菜单，**点一条指令**。  
3. 等它生成完，再追问「展开第二点」「改成表格」等。

相关订阅建议先打开 **自动 AI 摘要**，再跑指令，效果通常更好。

## 自定义指令（给进阶用户）

当 22 条不够用时，可以在设置里添加自己的指令，例如：「用三句话给老板汇报」「只提取涉及某公司的段落」。

### 要填哪些字段

| 字段 | 填什么 | 举例 |
|------|--------|------|
| **指令名称** | 菜单上显示的短名字 | `老板三句话汇报` |
| **提示词模板** | 详细告诉 AI：你是谁、要读什么、输出必须长什么样、禁止什么 | 见下方模板思路 |
| **上下文开关** | 是否自动带上摘要 / 对话历史 | 一般建议开摘要 |

### 模板里可用的变量

写提示词时可以插入这些占位符，运行时会自动换成真实内容：

| 变量 | 会被替换成 |
|------|------------|
| `{{summary_data}}` | 当前文章的 AI 摘要和标题等基础信息 |
| `{{original_context}}` | 原文内容（太长时系统可能会截断） |
| `{{history}}` | 多步任务里，上一步 AI 已经写出来的内容 |

**写自定义指令的笨办法（但有效）：** 在模板里固定写清四件事——

1. 角色（你是谁）  
2. 输入（你要读变量里的哪些内容）  
3. 输出格式（必须有哪些小标题 / 表格）  
4. 禁止事项（不要编造链接、不要空话等）  

### 自定义示例（可直接模仿）

```text
你是面向忙碌管理者的汇报助手。
请只根据 {{summary_data}} 写作。
输出必须包含：
1) 一句话结论
2) 三条要点（每条不超过 25 字）
3) 一个需要我决策的问题
禁止：不要用「赋能」「闭环」等空话；不要编造文中没有的数据。
```

## 常见问题

**Q：点了指令没反应或很空？**  
A：多半是当前对话里没有载入文章。请从信息流分组进聊天，或确认上下文里已有文章。

**Q：输出很像正确的废话？**  
A：先确认自动摘要已完成；换更匹配的指令（例如争议事件用「事实简报」而不是「推特长文」）；或换更强的「复杂模型」。

**Q：和专家指令集有什么区别？**  
A：能力相同。专家指令集把 22 条写细并讲组合玩法；本页解决「从哪点、每条白话含义、怎么自定义」。

### English

**AI Commands** let you one-tap specialist roles in chat instead of writing long prompts.

- 22 built-ins (some need **Pro**).  
- Full deep catalog + combos: **AI Smart Core → Expert Command Set**.  
- This page: where to open, plain-language what each command does, how to customize.

## Where to open

Command icon near the chat input; Command Center in some side panels; open chat from a Flow group first so articles are loaded; manage customs under Settings.

**Three steps:** load article context → open menu → tap a command → follow up as needed. Enable auto-summary on feeds first when possible.

## Custom commands

| Field | Meaning |
|-------|---------|
| Name | Short label in the menu |
| Prompt template | Full instructions: role, inputs, required output shape, bans |
| Context toggles | Attach summary / history |

### Variables

| Variable | Becomes |
|----------|---------|
| `{{summary_data}}` | Summaries + basic metadata |
| `{{original_context}}` | Source text (may truncate) |
| `{{history}}` | Previous step output in multi-step flows |

**Template pattern:** role → inputs → required sections → forbidden behaviors.

### Example

```text
You are a briefing assistant for busy managers.
Use only {{summary_data}}.
Output must include:
1) one-sentence conclusion
2) three bullets (≤25 words each)
3) one decision question for me
Do not use empty buzzwords; do not invent numbers not in the sources.
```

## FAQ

**Empty/weak output?** Load articles into context; wait for auto-summary; pick a command that matches the material; try a stronger Advanced model if needed.
