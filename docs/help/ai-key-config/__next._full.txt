<h2>配置步骤</h2>
<ol>
<li><strong>选择供应商</strong>：在供应商网格中点击图标。</li>
<li><strong>填写 API Key</strong>：按供应商要求输入密钥（密码输入框）。</li>
<li><strong>填写 API Host</strong>（可选）：默认使用官方地址，可改为兼容中转。</li>
<li><strong>填写模型名称</strong>：手动输入，或点击输入框旁按钮拉取模型列表后选择。</li>
<li><strong>高级参数</strong>（可选）：展开 <strong>显示生成参数</strong>，调整 Temperature（默认 0.7）、Top P（默认 0.9）、Max Tokens。</li>
<li><strong>验证并暂存</strong>：点击 <strong>验证并暂存配置</strong>，后台发送测试请求。</li>
<li><strong>设为当前启用</strong>：验证成功后点击 <strong>设为当前启用供应商</strong>，配置才会真正生效。</li>
</ol>
<blockquote><p>仅「验证并暂存」不会生效；必须再点 <strong>设为当前启用</strong>。</p></blockquote>
<h2>本地模型配置</h2>
<p>选择 Local LLM 后，API Host 下方会显示 <strong>Ollama 跨域配置指引</strong>：</p>
<ul>
<li><strong>Ollama（桌面版）</strong>：设置环境变量 <code>OLLAMA_ORIGINS=chrome-extension://*</code></li>
<li><strong>Ollama（命令行）</strong></li>
<li>PowerShell：<code>$env:OLLAMA_ORIGINS=&quot;chrome-extension://*&quot;; ollama serve</code></li>
<li>CMD：<code>set OLLAMA_ORIGINS=chrome-extension://* &amp;&amp; ollama serve</code></li>
<li><strong>LM Studio</strong>：在设置中开启跨域访问</li>
</ul>
